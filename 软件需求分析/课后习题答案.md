## 第1章 面向对象技术概述

### 考法概述

本章主要考查面向对象的基本概念，包括对象、类、封装、继承、多态等核心概念，以及面向对象方法与结构化方法的区别。

### 一、填空题

1. 面向对象方法至少应当包含4个方面：**标识**、**分类**、**多态**和**继承**。

2. "面向对象"是把一组对象中的数据结构和行为**紧密**结合在一起组织系统的一种策略。

3. **标识**是一个对象可识别的特性。

4. **服务**是用来描述对象动态特征（行为）的一个操作序列。

5. **封装**就是把对象的属性服务结合成为一个独立的系统单位，并尽可能隐蔽对象的内部细节。

6. **消息**就是向对象发出的服务请求，它应该含有下述信息：提供服务的**对象标识**、服务标识、**输入信息**和回答信息。

7. 结构化分析方法侧重于对系统进行**功能分解**，面向对象分析方法侧重于从**现实对象**的角度出发去研究和理解问题。

8. 面向对象的基本建模原则：**抽象**、**封装**、**继承**和**分类**。

9. 面向对象的软件工程包括**面向对象分析（OOA）**、**面向对象设计（OOD）**、**面向对象编程（OOP）**、**面向对象测试（OOT）**和**面向对象的软件维护（OOSM）**等主要内容。

10. RDD方法，又称为**CRC（类、责任、协作）方法**。

11. 对象的概念是**对问题域中某个实体的抽象**，类的概念是**对具有相同属性和行为的一个或多个对象的描述**。

12. 类和对象的关系是**对象是类的实例**，**类是对象的定义模板**。

13. 属性的定义是**描述对象静态特征的一个数据项**，服务的定义是**描述对象动态特征（行为）的一个操作序列**。

14. 类属性的定义是**描述类的所有对象的共同特征的一个数据项。对于任何对象实例，它的属性值都是相同的**。

15. 类的定义包含**名称**、**属性**和**操作**三要素。

16. 面向对象程序设计的三大特性是**封装**、**继承**和**多态**。

17. 面向对象方法中的**继承**机制使得子类可以自动地拥有（复制）父类的全部属性和操作。

18. 面向对象技术采用以类为中心的**封装**、**继承**、**多态**等特性不仅支持软件复用，而且使软件维护工作可靠有效，可实现软件系统的柔性制造。

19. 面向对象的系统分析要确立**对象模型**、**行为模型**和**功能模型**3个系统模型。

2. 行为建模就是通过状态描述和导致系统改变状态的事件来显示系统的行为。（**√**）
3. 结构化分析SA侧重于对系统进行功能分解。（**√**）
4. 面向对象分析OOA侧重于从现实对象的角度出发来分析系统。（**√**）
5. 可以使用Pascal编程语言进行面向对象的编程OOP。（**×**）
6. 封装就是把对象的属性和行为组成一个独立的系统单位，并且不用担心对象内部细节的隐蔽。（**×**）
7. 继承对于软件重用有着重要的意义，一般类继承特殊类，本身就是软件重用。（**×**）

### 四、简答题

**1. 叙述对象和类的关系。**

答：对象是类的实例，类是对象的模板。类是对具有相同属性和行为的一组对象的抽象描述，对象是类的具体实例。

**2. 简述面向对象的概念。**

答：Peter Coad和Edward Yourdon提出用以下公式来认识面向对象方法：
> 面向对象 = 对象 + 类 + 继承 + 消息传递

- **对象**：系统中用来描述客观事物的一个实体，由一组属性和一组行为构成
- **类**：具有相同属性和行为的一组对象的集合
- **继承**：特殊类的对象拥有其一般类的全部属性和行为
- **消息传递**：对象间通信的手段

**3. 简述面向对象设计的原则。**

答：面向对象设计原则有6个：开放封闭原则、单一职责原则、依赖倒置原则、Liskov替换原则、迪米特法则、接口隔离原则。

**4. 对象之间如何协同工作？**

答：对象之间通过消息进行通信，一个对象通过向另一个对象发送消息来请求其服务。

**5. 简述封装与信息隐藏的关系。**

答：封装是把对象的属性和服务结合成一个独立的系统单位，并尽可能隐蔽对象的内部细节。信息隐藏是封装的核心思想，即隐藏对象的内部实现细节，只暴露必要的接口。

**6. 简述多态性说明了什么。**

答：多态性是指在一般类中定义的属性或操作，在特殊类中可以具有不同的数据类型或表现出不同的行为。

**7. 简述重载与覆盖的区别。**

答：
- **重载（Overload）**：同一个类中多个同名方法，参数列表不同
- **覆盖（Override）**：子类重写父类的同名方法，参数列表相同

### 五、简单分析题

**某银行办理取款手续的需求描述如下：**

问题陈述：储户把存折和取款单一并交给银行柜台出纳员检验。出纳员核对账目，一旦发现存折有效性问题、取款单填写问题、账目与取款单不符等问题，出纳员停止本次取款活动，并将原因告知储户。在检查通过后，出纳员将取款信息登记在存折和账目上，并将现金交付给储户。

根据上述需求，完成以下任务：
1. 确定"取款"用例的描述
2. 识别对象
3. 识别对象的属性
4. 识别对象的行为，确定并画出"取款"用例的状态-事件-响应图
5. 确定对象所属的类，定义类的结构

**解答要点：**
- "取款"用例描述：①出纳员检验存折，核对账目；②如果出现问题则停止取款；③检查通过后登记信息并交付现金
- 识别对象：存折、账目（通过筛选，拒绝储户、取款单、银行柜台、出纳员、取款信息、现金等）
- 存折属性：存折号码、户主姓名、开户日期、存折类型等
- 账目属性：存折号码、交易类型、交易金额、交易日期等

10. 状态图和活动图在工具的支持下，可以实现代码的自动生成。（**√**）

### 四、简答题

**1. 简述状态图和活动图之间的区别。**

答：
- **状态图**：描述一个对象在其生命周期里随事件的发生而改变状态的变化过程，强调对象的状态变化
- **活动图**：描述系统从一个活动到另一个活动的流程，强调系统的业务流程或操作流程

**2. 状态图中一定要有终止状态吗？举例说明。**

答：不一定。例如，一个持续运行的系统（如操作系统）的状态图可能没有终止状态。

**3. 简述分支和分叉的区别。**

答：
- **分支**：将转换路径分成多个部分，每一部分都有单独的监护条件和不同的结果，是互斥的
- **分叉**：将一个控制分解为并行运行的并发线程，是并发的

**4. 泳道的作用是什么？**

答：泳道用于对活动的职责进行组织，将活动状态分为不同的组，每一组表示一个特定的类、人或部门，它们负责完成组内的活动。

**5. 什么是子状态？什么是组合状态？**

答：
- **子状态**：嵌套在另一个状态内部的状态
- **组合状态**：包含子状态的状态，可以进一步细化

## 第9章 面向对象设计原则

### 考法概述

本章考查面向对象设计的六大原则，以及每个原则的含义和应用场景。

### 核心知识点

1. **单一职责原则（SRP）**：一个类应该只有一个引起它变化的原因
2. **开放封闭原则（OCP）**：对扩展开放，对修改关闭
3. **里氏替换原则（LSP）**：子类必须能够替换其父类
4. **依赖倒置原则（DIP）**：依赖于抽象，不依赖于具体
5. **接口隔离原则（ISP）**：客户端不应该依赖它不需要的接口
6. **迪米特法则（LoD）**：一个对象应该对其他对象有最少的了解

| 组合 | 整体-部分关系（强） | 不可分离 |
| 依赖 | 一个类的变化影响另一个类 | 临时性 |
| 泛化 | 一般-特殊关系 | 继承性 |
| 实现 | 接口与实现类之间的关系 | 抽象性 |
