# 计算机组成原理作业三详细解析

## 题目一：流水线性能分析

### 题目描述
假设某型机的 CPU 指令周期有五个功能段，分别为取指令 IF、译码并读寄存器 ID、执行指令 EX、存储器访问（MEM）、将结果存回寄存器 WB。流水线中各段的执行时间分别是 4ns、5ns、6ns、7ns、4ns。若该机使用流水线执行 200 条指令，假设不存在冲突问题。

### 知识点回顾
1. **流水线基本概念**：将指令执行过程分成多个阶段，每个阶段由专门的硬件完成，不同指令的不同阶段可以重叠执行。
2. **流水线性能指标**：
   - **吞吐率（Throughput）**：单位时间内完成的任务数量
   - **加速比（Speedup）**：非流水线执行时间与流水线执行时间的比值
   - **效率（Efficiency）**：流水线中各段实际使用时间与总时间的比值

### 解题步骤

#### (1) 流水线的最大吞吐率

**关键概念**：流水线的最大吞吐率取决于最慢的那个阶段（瓶颈段）。

**计算过程**：
- 各段执行时间：IF=4ns, ID=5ns, EX=6ns, MEM=7ns, WB=4ns
- 瓶颈段是 MEM 段，执行时间为 7ns
- 最大吞吐率 TP_max = 1 / 瓶颈段时间 = 1 / 7ns

**答案**：
```
TP_max = 1 / 7ns ≈ 142.86 条指令/秒
```

#### (2) 执行完成 200 条指令需要的时间

**关键概念**：流水线执行时间 = 第一条指令完成时间 + (n-1) × 瓶颈段时间

**计算过程**：
- 第一条指令需要经过所有 5 个阶段：4 + 5 + 6 + 7 + 4 = 26ns
- 剩余 199 条指令每隔 7ns 完成一条
- 总时间 = 26ns + (200-1) × 7ns = 26ns + 1393ns = 1419ns

**答案**：
```
T = 26 + (200-1) × 7 = 26 + 1393 = 1419 ns
```

#### (3) 流水线的效率

**关键概念**：效率 = 实际使用时间 / (段数 × 总时间)

**计算过程**：
- 段数 k = 5
- 总时间 T = 1419ns
- 实际使用时间 = 200 条指令 × 每条指令 5 个阶段 × 各阶段平均时间
  - 更简单的方法：效率 = (n × Σt_i) / (k × T)
  - 其中 Σt_i = 4+5+6+7+4 = 26ns（一条指令的总执行时间）
- 效率 = (200 × 26) / (5 × 1419) = 5200 / 7095 ≈ 0.733

**答案**：
```
效率 η = (200 × 26) / (5 × 1419) = 5200 / 7095 ≈ 73.3%
```

#### (4) 指令序列相关性分析

**指令序列**：
```
MUL R2, R1, R0  ; (R1)×(R0)→R2
JEQ X           ; 乘法结果为零则转移
ADD R3, R4, R5  ; (R4)+(R5)→R3
```

**相关类型分析**：
1. **数据相关（Data Hazard）**：
   - MUL 指令计算结果存入 R2
   - JEQ 指令需要检查乘法结果（通常检查零标志或结果寄存器）
   - 如果 JEQ 依赖 MUL 的结果，则存在数据相关

2. **控制相关（Control Hazard）**：
   - JEQ 是条件转移指令
   - 转移是否发生取决于 MUL 的结果
   - 这会导致流水线控制冲突

**具体分析**：
- MUL 和 JEQ 之间存在**数据相关**：JEQ 需要 MUL 的结果来判断是否转移
- MUL 和 ADD 之间**无相关**：ADD 使用的寄存器 R3、R4、R5 与 MUL 使用的 R1、R2、R0 不同
- JEQ 和 ADD 之间存在**控制相关**：如果 JEQ 发生转移，ADD 可能不会执行

**答案**：
```
存在相关：
1. 数据相关：MUL → JEQ（JEQ 依赖 MUL 的结果）
2. 控制相关：JEQ → ADD（转移指令影响后续指令执行）
```

## 题目二：微程序设计

### 题目描述
假设 CPU 采用非总线结构（如图所示），微程序设计中每个机器周期执行一条微指令（每个机器周期包含 3 个节拍）。

### 图片分析（CPU 结构图）
从图片可以看出这是一个典型的 CPU 数据通路，包含以下主要部件：
- **M（存储器）**：主存储器
- **MAR**：存储器地址寄存器
- **MDR**：存储器数据寄存器
- **PC**：程序计数器
- **IR**：指令寄存器
- **AC**：累加器
- **ALU**：算术逻辑单元
- **CU**：控制单元
- **时钟**：提供时序信号

### 知识点回顾
1. **微操作**：在一个时钟周期内完成的最基本操作
2. **节拍**：时钟周期的时间单位
3. **机器周期**：一个完整的指令执行阶段，包含多个节拍
4. **微指令**：一组同时执行的微操作

### 解题步骤

#### (1) 取指阶段的微操作及节拍安排

**取指阶段的任务**：
1. 从存储器取出指令
2. 送入指令寄存器 IR
3. PC 自增指向下一条指令

**微操作分析**：
- 第 1 拍：PC → MAR（将 PC 的值送到 MAR，准备读取指令）
- 第 2 拍：M(MAR) → MDR，PC + 1 → PC（从存储器读取指令，同时 PC 自增）
- 第 3 拍：MDR → IR（将指令从 MDR 送到 IR）

**答案**：
```
取指阶段微操作：
T0: PC → MAR
T1: M(MAR) → MDR, PC + 1 → PC
T2: MDR → IR
```

#### (2) 各指令执行阶段的微操作及节拍安排

##### STA X（将累加器 ACC 的数据写入主存 X 地址单元）

**任务分析**：
1. 从 IR 中取出地址 X
2. 将 AC 的数据写入存储器

**微操作分析**：
- 第 1 拍：IR(地址) → MAR（将指令中的地址 X 送到 MAR）
- 第 2 拍：AC → MDR（将累加器的数据送到 MDR）
- 第 3 拍：MDR → M(MAR)（将数据写入存储器）

**答案**：
```
STA X 执行阶段微操作：
T0: IR(地址) → MAR
T1: AC → MDR
T2: MDR → M(MAR)
```

##### AND X（将主存 X 地址单元的数据与累加器 ACC 的数据按位与，结果送 ACC）

**任务分析**：
1. 从存储器读取数据
2. 与 AC 进行按位与运算
3. 结果存回 AC

**微操作分析**：
- 第 1 拍：IR(地址) → MAR（将地址 X 送到 MAR）
- 第 2 拍：M(MAR) → MDR（从存储器读取数据）
- 第 3 拍：AC ∧ MDR → AC（进行按位与运算，结果存回 AC）

**答案**：
```
AND X 执行阶段微操作：
T0: IR(地址) → MAR
T1: M(MAR) → MDR
T2: AC ∧ MDR → AC
```

##### ADD #a（将立即数 a 与累加器 ACC 的数据相加，结果送 ACC）

**任务分析**：
1. 立即数 a 已经在指令中（IR 的地址字段）
2. 与 AC 进行加法运算
3. 结果存回 AC

**微操作分析**：
- 第 1 拍：IR(地址) → MDR（将立即数 a 从 IR 送到 MDR）
- 第 2 拍：AC + MDR → AC（进行加法运算，结果存回 AC）
- 第 3 拍：空操作（或用于其他控制）

**注意**：立即数寻址不需要访问存储器，所以只需要 2 个节拍。

**答案**：
```
ADD #a 执行阶段微操作：
T0: IR(地址) → MDR
T1: AC + MDR → AC
T2: 空操作
```

## 总结与学习要点

### 流水线相关
1. **瓶颈段决定性能**：流水线的最大吞吐率由最慢的阶段决定
2. **执行时间计算**：第一条指令时间 + (n-1) × 瓶颈段时间
3. **效率计算**：实际使用时间与理论最大时间的比值
4. **相关性分析**：注意数据相关和控制相关

### 微程序设计相关
1. **理解数据通路**：根据 CPU 结构图分析数据流动路径
2. **微操作顺序**：按照任务需求安排微操作的执行顺序
3. **节拍分配**：每个节拍完成一个微操作，注意并行性
4. **寻址方式影响**：立即数寻址不需要访问存储器，节省节拍

### 解题技巧
1. **画图辅助**：对于复杂的数据通路，画出数据流动图有助于理解
2. **分步分析**：将复杂任务分解为简单的微操作
3. **检查相关性**：分析指令之间是否存在数据依赖或控制依赖
4. **验证答案**：检查微操作序列是否完整实现了指令功能
