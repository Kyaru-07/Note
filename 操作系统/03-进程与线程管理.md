# 第5讲 进程控制

## 1. 概念引入

### 1.1 并发与逻辑流

- **并发（Concurrency）**：多个活动在同一时间段内同时进行，是操作系统最基本的特征。体现为多任务、多进程、多线程的并发执行。
- **逻辑控制流（Logical Control Flow）**：在多进程环境下，一个进程从 main 函数开始，一条条指令执行，其执行轨迹为一系列程序计数器（PC）的值，给人一种每个程序都独占处理器的假象。实际上各进程交替使用处理器。
- **并发流（Concurrent Flow）**：两个逻辑流的执行在时间上可能存在重叠，则它们是并发的。判断准则：只要某种可能的执行模式在时间上存在重叠，它们就是并发流。
- **并行流（Parallel Flow）**：两个流在某段时间内同时运行在不同的处理器核或计算机上。并行流是并发流的一个真子集。

### 1.2 多任务与时间片

- **多任务（Multitasking）**：一个进程和其他进程轮流运行的概念。
- **时间片（Time Slice）**：每次分配给一个进程的执行时间。进程因此被划分为多个时间分片。
- CPU 控制转移时机：一般在中断响应之时（外部中断、时钟中断）。
- 进程主动放弃 CPU（如 I/O 操作）或被动放弃 CPU（时间片用完、高优先级抢占）。

### 1.3 进程的定义

- **非严格定义**：进程是正在执行中的程序。
- **严格定义**：进程是程序在一个独立数据集上的执行过程。
- Linux 允许多个用户同时登录，每个用户可同时运行多个程序，每个程序实例都是一个进程。系统本身也运行管理系统资源的进程。

### 1.4 进程的结构

进程至少包含三项内容：

| 组成部分 | 说明 |
|---------|------|
| **程序代码** | 包括 main 函数的可执行程序，装载到内存后进程才能启动 |
| **数据集** | 进程的处理对象（变量内容），包括初始化数据、环境变量、命令行参数和文件数据 |
| **进程控制块（PCB）** | 保存程序代码、数据变量地址及其他属性。PCB 是进程存在的**唯一标志**，操作系统通过 PCB 对进程实施管理和控制 |

### 1.5 进程属性（保存在 PCB 中）

1. **进程描述信息**：PID、UID、GID、族亲信息（父进程、兄弟进程）
2. **进程控制信息**：进程状态、调度信息（优先级、剩余时间片、调度策略）、记时信息、通信信息
3. **进程资源信息**：存储器地址、打开文件描述符表
4. **CPU 现场信息（进程上下文）**：程序计数器 PC、通用寄存器、标志寄存器 FLAGS

### 1.6 PCB 实例：Linux 的 task_struct

```c
struct task_struct {
    pid_t pid;                    // 进程识别码
    uid_t uid, euid, suid, fsuid; // 用户标识码
    gid_t gid, egid, sgid, fsgid; // 用户组标识码
    struct task_struct *p_opptr, *p_pptr, *p_cptr,
         *p_ysptr, *p_osptr;      // 父子兄弟进程指针
    struct task_struct *prev_task, *next_task; // 进程队列指针
    volatile long state;          // 进程状态
    long counter;                 // 剩余时间片
    long priority;                // 优先级
    unsigned long policy, rt_priority; // 调度策略，实时优先级
    struct mm_struct *mm;         // 存储器资源
    struct fs_struct *fs;         // 打开文件资源
    ...
}
```

### 1.7 进程的基本状态及转换

三种基本状态：

- **就绪状态（Ready State）**：已分配除 CPU 外的所有必要资源，只要获得 CPU 便可立即执行。
- **运行状态（Running State）**：已获得包括 CPU 在内的所有资源，正在执行。
- **阻塞状态（Blocked State）**：因请求资源、等待事件或等待 I/O 等原因暂时无法继续执行。

另外两种状态：
- **创建状态**：正在创建且尚未完成创建过程。
- **终止状态**：进程终止后并不立即将其清理，保留 PCB 等待父进程读取。

### 1.8 进程 PCP 组织

Linux 以多种方式组织进程：

1. **双向链表队列**：按进程状态组织为多个双向链表。设置一个就绪队列和多个阻塞队列，每种等待事件一个等待队列。
2. **双向链表 + 树形结构**：树形结构表示父子关系（p_pptr、p_cptr），子进程间构成双向链表（p_osptr、p_ysptr）。

### 1.9 进程与程序的区别

| 方面 | 程序 | 进程 |
|------|------|------|
| 存在性 | 永久的，存在于外存 | 暂时的，有创建有撤销 |
| 静动态 | 静态的，关机后仍存在 | 动态的，有生命周期 |
| 并发性 | 没有并发性 | 具有并发性 |
| 对应关系 | 一个程序可对应多个进程 | 一个进程可执行一个或几个程序 |

## 2. 核心公式

### 2.1 并发进程可能的执行顺序

若两个并发进程 P1 有 m 个活动、P2 有 n 个活动，且各活动以不可分割方式执行，则不同的执行顺序共有 **C(m+n, m)** 种（即从 m+n 个位置中选 m 个给 P1 的组合数）。

### 2.2 进程回收相关宏

```c
WIFEXITED(status)    // 子进程正常终止返回真
WIFSIGNALED(status)  // 子进程因信号终止返回真
WEXITSTATUS(status)  // 返回正常终止子进程的退出状态
```

## 3. 方法速通

### 3.1 进程创建：fork()

```c
#include <sys/types.h>
#include <unistd.h>
pid_t fork(void);
```

- **返回值**：子进程返回 0，父进程返回子进程的 PID，出错返回 -1。
- **fork 执行结果**：创建父进程 PCB 和数据集的副本给子进程；子 PCB 的 PID 为新分配的 PID；父进程 fork 返回子进程 PID，子进程返回 0。

### 3.2 进程终止

**正常终止**：完成 main 函数执行、在 main 中 return、执行 exit() 调用。

**异常终止**：执行 abort()、按 Ctrl_C、程序出错（被 0 除、存储器访问越界、数学运算溢出）、被信号终止。

```c
#include <stdlib.h>
void exit(int status);   // 正常终止，终止状态为 status
void abort(void);        // 异常终止，终止状态为 6
```

### 3.3 进程回收

```c
pid_t waitpid(pid_t pid, int *status, int options);
// 成功返回子进程 PID，出错返回 -1
// pid=-1: 等待任意子进程；options=0: 阻塞等待
```

### 3.4 进程休眠

```c
unsigned int sleep(unsigned int secs);  // 睡眠若干秒
void usleep(unsigned int usecs);        // 睡眠若干微秒
int pause(void);                        // 无限期休眠，直到收到信号
```

### 3.5 进程加载（exec 系列）

```c
int execve(const char *filename, const char *argv[], const char *envp[]);
int execvp(const char *filename, const char *argv[]);
int execlp(const char *file, const char *arg, ...);
// 成功不返回，出错返回 -1
```

### 3.6 进程属性读取

```c
pid_t getpid(void);   // 返回当前进程 PID
pid_t getppid(void);  // 返回父进程 PID
uid_t getuid(void);   // 返回实际用户 ID
uid_t geteuid(void);  // 返回有效用户 ID
gid_t getgid(void);   // 返回实际用户组 ID
gid_t getegid(void);  // 返回有效用户组 ID
```

## 4. 典型示例

### 4.1 fork 基本示例

```c
#include "wrapper.h"
int main() {
    pid_t pid;
    int x = 1;
    pid = fork();
    if (pid == 0) {       // 子进程
        x = x + 1;
        printf("child: x=%d\n", x);   // 输出 x=2
    }
    if (pid > 0) {        // 父进程
        x = x - 1;
        printf("parent: x=%d\n", x);  // 输出 x=0
    }
    sleep(10);
}
```

- 父子进程各自拥有独立的 x 变量副本，互不影响。

### 4.2 多次 fork 的族亲关系

```c
#include <unistd.h>
int main() {
    int pid;
    pid = fork();
    pid = fork();
    if (pid > 0) fork();
    printf("hello\n");
    exit(0);
}
```

共产生 7 个进程（包括原始进程）。通过 `ps -ef | grep fork2s` 可验证进程树关系。

### 4.3 编写多进程并发程序

根据给定的进程族亲关系图，利用 fork() 和 if 判断创建多个子进程，各进程打印自己的 PID。

### 4.4 多进程并发特征示例

```c
int main() {
    pid_t pid;
    pid = Fork();
    if (pid > 0) {
        A: printf("a");
        B: printf("b");
        exit(0);
    } else {
        C: printf("c");
        D: printf("d");
        exit(0);
    }
}
// 可能的输出顺序有 6 种：abcd, acbd, acdb, cdab, cadb, cabd
```

### 4.5 进程回收示例

```c
#define N 2
int main() {
    int status, i;
    pid_t pid[N], retpid;
    for (i = 0; i < N; i++)
        if ((pid[i] = fork()) == 0)
            exit(100 + i);
    while ((retpid = waitpid(-1, &status, 0)) > 0) {
        if (WIFEXITED(status))
            printf("child %d terminated normally with exit status=%d\n",
                   retpid, WEXITSTATUS(status));
        else
            printf("child %d terminated abnormally\n", retpid);
    }
    if (errno != ECHILD)
        perror("waitpid error");
    exit(0);
}
```

### 4.6 exec 程序加载示例

```c
// 用 execvp 加载并运行 /bin/ps 命令
#include "wrapper.h"
int main() {
    char *argv[] = {"ps", "-o", "pid,ppid,pgrp,session,tpgid,comm", NULL};
    if (fork() == 0) {
        execvp("/bin/ps", argv);
        // 若 exec 成功则不返回
    }
    wait(NULL);
    exit(0);
}
```

### 4.7 fork + exec + wait 实现简易 Shell

```c
int main() {
    char cmdline[MAXLINE];
    while (1) {
        printf("%% ");
        fgets(cmdline, MAXLINE, stdin);
        if (feof(stdin)) exit(0);
        eval(cmdline);   // 分析并执行命令
    }
}
```

流程：输入命令行 -> 解析命令行 -> 判断是否内部命令 -> 创建子进程 -> 加载命令(exec) -> 等待命令完成(waitpid)。

### 4.8 非本地跳转示例

```c
jmp_buf buf;
int main() {
    int rc;
    rc = setjmp(buf);
    if (rc == 0) foo();
    else if (rc == 1) printf("Detected error1 in foo\n");
    else if (rc == 2) printf("Detected error2 in foo\n");
    exit(0);
}
void foo(void) {
    if (error1) longjmp(buf, 1);
    bar();
}
void bar(void) {
    if (error2) longjmp(buf, 2);
}
```

### 4.9 僵尸进程示例

```c
int main() {
    pid_t pid;
    pid = Fork();
    if (pid == 0) {
        printf("child...\n");
    } else {
        printf("parent...\n");
        while(1);  // 父进程死循环，不调用 waitpid
    }
    return 0;
}
// 子进程终止后变为僵尸进程（Z+ 状态），其 PCB 仍然保留
```

## 5. 真题/实战

### 5.1 进程族亲关系分析

**题目**：分析 fork2.c 程序的进程族亲关系和输出。

```c
int main() {
    int pid;
    pid = fork();
    pid = fork();
    if (pid > 0) fork();
    printf("hello\n");
    exit(0);
}
```

**分析**：共产生 7 个进程，进程树关系为：原始进程产生两个直接子进程，第二个 fork 产生的子进程（pid>0）又产生一个子进程。输出 7 个 "hello"。

### 5.2 并发输出顺序分析

**题目**：fork5.c 中两个进程分别打印 a,b 和 c,d，可能的输出顺序有哪些？

**答案**：6 种——abcd, acbd, acdb, cdab, cadb, cabd。

### 5.3 exec 执行后代码不可达

```c
if (pid == 0) {
    execlp("/bin/ls", "ls", NULL);
    printf("LINE J");  // exec 成功后不返回，此行只有 exec 失败才执行
}
```

### 5.4 信号处理与进程并发竞争

**题目**：父进程创建子进程后，子进程通过 exec 执行程序，父进程需要 addjob 将子进程加入作业列表。但子进程可能在父进程 addjob 之前就终止并发送 SIGCHLD 信号。

**解决**：在 fork 之前阻塞 SIGCHLD，addjob 之后再解除阻塞，保证 addjob 在 deletejob 之前执行。

# 第6讲 线程管理与同步互斥

## 1. 概念引入

### 1.1 线程的定义

- **线程（Thread）**：进程内一个执行单元或一个可调度实体，每个执行单元可执行进程的一段程序代码（如函数）。一个进程内可创建多个线程，它们都是并发逻辑流。
- 线程共享进程资源：同一进程的多个线程位于同一地址空间，共享代码、数据（全局变量）、堆和打开文件等。
- 线程又称**轻量级进程（LWP）**，创建、销毁开销远低于进程（不到进程开销的 10%）。

### 1.2 传统进程的不足

- 一个进程只有一个代码执行序列（控制流），只能接受一个 CPU 服务，无法发挥多 CPU/多核能力。
- 进程每段时间只能执行一项活动，难以同时协调多项并发活动。

### 1.3 线程结构与状态

- 每个线程拥有一个**线程控制块 TCB（Thread Control Block）**和一个线程堆栈。
- TCB 内容：线程 ID（TID）、线程状态、栈指针、通用寄存器、程序计数器、标志寄存器。
- 线程基本状态：就绪、运行、阻塞。

### 1.4 线程执行模型

- **主线程（Main Thread）**：进程启动时执行 main 函数的线程。
- **对等线程（Peer Thread）**：主线程创建的新线程，之后各线程可继续创建更多对等线程。
- 线程之间**无父子关系**，都是对等的，组成一个**对等线程池（Pool）**。
- 一个线程可以终止任何其他对等线程，或等待任意其他对等线程终止。

### 1.5 多线程程序设计的好处

- 让程序同时做多件事情（如编辑文档同时统计字数）。
- 将耗时任务划分成多小任务（如大矩阵运算）。
- 管理多项并发活动（如网络服务器同时处理多个连接）。

---

## 2. 核心公式

### 2.1 共享变量识别准则

一个变量是**共享的**，当且仅当它的一个实例被一个以上的线程引用。

可能的共享变量：
1. 全局变量
2. 静态变量
3. 其地址通过参数或全局变量传递给其他线程的局部变量

### 2.2 并行程序加速比

$$T_n = T_S + \frac{T_P}{n} + n \cdot t$$

其中：
- $T_S$ = 不可并发部分在单处理器上的执行时间
- $T_P$ = 可并发部分在单处理器上的执行时间
- $n$ = 处理器数量（线程数）
- $t$ = 每个线程因互斥必须串行执行的时间

**加速比（Speedup）**：$S = T_1 / T_n$

**效率（Efficiency）**：$E = S / n$（超过 90% 非常好）

## 3. 方法速通

### 3.1 Pthreads 线程 API

```c
// 创建线程
int pthread_create(pthread_t *tid, pthread_attr_t *attr,
                   Func fun, void *arg);

// 获得线程 TID
pthread_t pthread_self(void);

// 终止线程（根据 tid 撤销）
void pthread_cancel(pthread_t tid);

// 线程自己终止
void pthread_exit(void *thread_return);

// 等待线程终止并回收资源
int pthread_join(pthread_t tid, void **thread_return);

// 分离线程（终止时自动释放资源）
int pthread_detach(pthread_t tid);

// 初始化线程状态（只执行一次）
int pthread_once(pthread_once_t *once_control,
                 void (*init_routine)(void));
```

### 3.2 线程间数据传递三种方式

1. 利用 pthread_create 的第四个参数（void *arg）
2. 通过全局变量
3. 通过 pthread_exit 函数参数（返回值）

### 3.3 信号量与 P/V 操作

**信号量（Semaphore）**：Dijkstra 提出的同步设施，本质是封装了加 1 和减 1 原语操作的计数器。

```c
typedef struct _semaphore {
    int value;        // 信号量值，表示可用资源数
    WaitQueue L;      // 阻塞队列
} semaphore;
```

- **P 操作（wait）**：信号量值减 1；若值 < 0，则进程阻塞
- **V 操作（signal）**：信号量值加 1；若值 <= 0，则唤醒一个阻塞进程

### 3.4 用信号量解决临界区互斥

```c
semaphore mutex = 1;  // 互斥信号量，初值为 1

void Thread() {
    wait(mutex);      // 进入区：获取许可权
    <<临界区>>        // 操作临界资源
    signal(mutex);    // 退出区：归还许可权
    <<剩余代码>>
}
```

### 3.5 用信号量解决同步问题

**单向同步**：ActionA 必须在 ActionB 之前完成。

```c
semaphore sem = 0;      // 虚拟资源，初始不存在

// ActionA（产生数据）
<<ActionA>>
signal(sem);             // 产生虚拟资源

// ActionB（消费数据）
wait(sem);               // 获取虚拟资源
<<ActionB>>
```

**双向同步**：A 写 buffer 前要获取空槽位，写完产生数据项；B 读 buffer 前要获取数据项，读完产生空槽位。

```c
semaphore avail = 1;    // 空槽位数
semaphore ready = 0;    // 数据项数

// 生产者
wait(avail);            // 获取空槽位
<<写 buffer>>
```

### 4.6 生产者/消费者问题

```c
#define N 20
int buf[N], outpos = 0, inpos = 0;
semaphore avail = N, ready = 0, mutex = 1;

void Thread_Pi() {  // 生产者
    while (1) {
        item = produce();
        wait(avail);       // 获取空槽位
        wait(mutex);       // 互斥访问缓冲区
        sbuf_insert(item, buf);
        signal(mutex);
        signal(ready);     // 产生数据项
    }
}

void Thread_Ci() {  // 消费者
    while (1) {
        wait(ready);       // 获取数据项
        wait(mutex);       // 互斥访问缓冲区
        item = sbuf_remove(buf);
        signal(mutex);
        signal(avail);     // 归还空槽位
        consume(item);
    }
}
```

**关键点**：
- avail（空槽位）初值 = N，ready（数据项）初值 = 0
- mutex 保证缓冲区操作互斥
- P 操作顺序：先 avail/ready，后 mutex（否则可能死锁）

### 4.7 读者/写者问题

```c
semaphore mutex = 1, wmutex = 1;
int count = 0;

Writer_i() {
    while (1) {
        wait(wmutex);
        <<写者操作>>
        signal(wmutex);
    }
}

Reader_i() {
    while (1) {
        wait(mutex);
        count++;
        if (count == 1)    // 第一个读者
            wait(wmutex);  // 获取写锁
        signal(mutex);
        <<读者操作>>
        wait(mutex);
        count--;
        if (count == 0)    // 最后一个读者
            signal(wmutex); // 释放写锁
        signal(mutex);
    }
}
```

**关键点**：
- wmutex 控制写者互斥和读写互斥
- mutex 保护 count 变量的互斥访问
- 第一个读者上锁，最后一个读者解锁

### 5.1 共享变量识别

**题目**：识别以下程序中的共享变量和非共享变量。

```c
char **ptr;  // 全局变量
int main() {
    int i;
    pthread_t tid;
    char *msgs[N] = {"Hello from foo", "Hello from bar"};
    ptr = msgs;
    for (i = 0; i < N; i++)
        Pthread_create(&tid, NULL, thread, (void *)i);
    Pthread_exit(NULL);
}
void *thread(void *vargp) {
    int myid = (int)vargp;
    static int cnt = 0;
    printf("[%d]: %s (cnt=%d)\n", myid, ptr[myid], ++cnt);
    return NULL;
}
```

**答案**：
- **共享变量**：ptr（全局）、cnt（静态）、msgs[0]、msgs[1]（通过 ptr 共享）
- **非共享变量**：myid（每个线程独立实例）、i、tid

### 5.2 卖票问题改进

**问题**：将 while 循环条件 tickets>0 放在锁外面，仍可能出现问题。

### 5.3 条件变量实现生产者消费者

```c++
queue<int> products;
mutex mtx;
condition_variable cvar;
bool done = false;

void produce() {
    for (int i = 1; i <= 5; i++) {
        this_thread::sleep_for(chrono::seconds(2));
        unique_lock<mutex> locker(mtx);
        cout << "生产产品" << i << " ";
        products.push(i);
        cvar.notify_one();
    }
    done = true;
    cvar.notify_one();
}

void consume() {
    unique_lock<mutex> locker(mtx);
    while (!done) {
        cvar.wait(locker);  // 阻塞等待
        while (!products.empty()) {
            cout << "消费产品" << products.front() << endl;
            products.pop();
        }
    }
}
```

## 7. 总结复盘

### 本讲术语

线程、变量实例、共享变量、临界资源、临界区、信号量、P/V 操作、竞争、线程安全函数、可重入函数、顺序程序、并发程序、并行程序、线程加速比、线程效率

### 核心知识点回顾

1. **线程定义**：进程内的执行单元，共享进程资源，拥有独立的 TCB 和堆栈
2. **线程执行模型**：主线程 + 对等线程池，线程间无父子关系
3. **共享变量识别**：全局变量、静态变量、地址通过参数传递的局部变量
4. **临界资源与临界区**：被多个并发流共用但需独占使用的资源及操作它的代码段
5. **信号量与 P/V 操作**：
   - 互斥信号量（初值 1）：保护临界资源
   - 同步信号量（初值 0 或 N）：协调执行顺序
6. **生产者/消费者问题**：3 个信号量（avail=N, ready=0, mutex=1），P 操作先 avail/ready 后 mutex
7. **读者/写者问题**：2 个信号量（wmutex=1, mutex=1），第一个读者上锁，最后一个读者解锁
8. **其他同步机制**：互斥量、条件变量、管程、原子类型、lock_guard
9. **线程安全**：四类不安全函数及改造方法
10. **并行性能**：加速比 S = T1/Tn，效率 E = S/n，减少互斥锁范围可提高性能
