# 第4讲 文件与磁盘管理

## 1. 概念引入

### 1.1 文件与文件系统

**文件（File）**：计算机系统管理数据信息的规范方法，系统中所有可存取的资料以文件形式存放在各目录下。文件为使用者提供操作数据的统一方法和界面，使用者无需了解存储设备种类和特性。进程调度、内存分配为进程服务，文件是并发进程的"原材料"与"输出品"。

**文件目录（Directory）**：使用目录可对大量文件进行分类、分层管理，方便检索、搜寻、操作文件。

### 1.2 核心问题引出

文件系统面临四个关键问题：
1. 如何分配磁盘块给文件（文件物理结构）
2. 空闲盘块如何管理（空闲盘块分配与管理）
3. 分解目录方案下基本目录如何管理（索引节点表）
4. 根目录作为文件系统入口放哪里（超级块）

## 2. 核心公式

### 2.1 位示图地址换算

已知位示图位置 (i, j)，对应磁盘块号：

```
n = 字大小 * i + j
```

已知磁盘块号 n，对应位示图位置：

```
i = INT(n / 字大小)
j = n % 字大小
```

### 2.2 目录分解性能对比

**一体化目录**：1000个目录项，每个48字节，盘块512字节
- 目录占盘块数 = (1000 x 48) / 512 = 94块
- 平均读盘次数 = (1 + 94) / 2 = **47.5次**

**分解式目录**：符号目录项8字节，基本目录项42字节
- 符号目录占盘块数 = (1000 x 8) / 512 = 16块
- 平均读盘次数 = 符号目录8.5次 + 基本目录1次 = **9.5次**（仅原来的20%）

### 2.3 多级索引最大文件大小

若盘块大小 B，每块可存地址数 A = B / 指针大小：

```
最大文件 = (直接地址数 + A + A^2 + A^3 + ...) * B
```

例：盘块4KB，指针4B，10个直接 + 1次间接 + 1次二级 + 1次三级
- 最大块数 = 10 + 1024 + 1024^2 + 1024^3 > 2^30
- 最大文件 = 2^30 x 4KB = **4TB**

### 2.4 FAT表开销计算

```
FAT占用空间 = 表项位宽(字节) x 簇数 x 副本数
簇数 = 磁盘容量 / 簇大小
```

例：256MB磁盘，簇4KB，FAT16，4个副本
- 簇数 = 256MB / 4KB = 64K
- FAT占用 = 2B x 64K x 4 = **512KB**（开销约0.2%）

### 2.5 混合索引逻辑地址计算

字节偏移量 offset 对应的逻辑块号：

```
逻辑块号 = offset / 块大小
```

根据逻辑块号判断寻址方式：
- 逻辑块号 < 直接地址数：直接寻址，读盘1次
- 逻辑块号落在一次间接范围：读索引块1次 + 数据块，共2次
- 逻辑块号落在二次间接范围：读二次索引块1次 + 一次索引块1次 + 数据块

## 3. 方法速通

### 3.1 文件逻辑结构判断

| 类型 | 特征 | 实例 |
|------|------|------|
| 无结构文件（流式文件） | 字节串，OS无特殊意义 | .txt, .c, .cpp, .java |
| 有结构文件（记录式文件） | 多条记录（定长/不定长） | .vcf通讯录, 学生登记表 |
| 复杂文件结构 | 文件头描述结构，魔数标识 | .exe, ELF, PDF |

**魔数（Magic Number）**：文件前4字节用于标识文件类型
- Windows PE：`0x00905A4D`
- Linux ELF：`0x464C457F`
- Nachos：`0x00badfad`

### 3.2 三种外存组织方式速查

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 连续组织 | 顺序访问速度快 | 删除/插入涉及大量移动 | 一次性写入的文件（如银行备份） |
| 隐式链接 | 增删效率高 | 随机访问效率低，链指针破坏则丢失 | 顺序存取为主 |
| 显式链接(FAT) | 随机访问高效（FAT可预读入内存） | FAT表开销，可靠性问题 | DOS/Windows |
| 索引组织 | 支持直接访问，无外部碎片 | 小文件索引块利用率低 | UNIX/Linux |

### 3.3 文件读写操作要点

**read(fd, buf, len)**：
1. 根据读写指针 pos 和长度 len，计算逻辑块号
2. 将数据块读到内存
3. 从数据块中选取数据复制到 buf

**write(fd, buf, len)**：
1. 计算逻辑块号
2. 若首末数据块非全部写入，先读入内存
3. 用 buf 更新首末数据块
4. 将数据块写回磁盘

### 3.4 空闲空间管理方法速查

| 方法 | 特点 | 适用 |
|------|------|------|
| 空闲表法 | 连续分配，适合外存连续分配 | 连续文件系统 |
| 空闲盘块链 | 简单但效率低 | 小型系统 |
| 空闲盘区链 | 可分配连续块，分配回收复杂 | 需要连续空间 |
| 位示图法 | 直观，适合中等规模 | 多数通用OS |
| 成组链接法 | UNIX标准方法，高效 | UNIX/Linux |

## 4. 典型示例

### 4.1 文件读写操作示例

**题目**：磁盘块大小512B，/home/can目录中文件f2的目录项如图。执行 `open("/home/can/f2", O_RDWR, 0)` 后：

**(1) 执行 lseek(fd,1000,SEEK_SET); read(fd,buf,500);**

解题过程：
- 逻辑块号 = 1000 / 512 = **1**（第2个逻辑块，从0计数）
- 字节在块内偏移 = 1000 % 512 = **488**
- 读取长度500字节，跨2个块（块1剩余24字节 + 块2前276字节）
- **读盘2次**：将逻辑块1和逻辑块2对应的实际磁盘块读入内存

**(2) 执行 lseek(fd,2000,SEEK_SET); write(fd,buf,500);**

解题过程：
- 首字节逻辑块号 = 2000 / 512 = **3**，末字节逻辑块号 = (2000+500-1) / 512 = **4**
- 需要写入逻辑块3和4
- 若块3非全部写入（2000在块3中间），需先**读盘1次**
- **写盘2次**：将块3和块4写回磁盘
- 共**读盘1次，写盘2次**

### 4.2 目录分解方案计算

**题目**：盘块512B，分解前目录项48B，分解后符号目录8B + 基本目录42B，/home/can含1000个文件。

**(1) f4内部号100，基本目录区起始盘块10000，求f4基本目录项盘块号**

```
字节距离 d = 42 x 100 = 4200
盘块偏移 = 4200 / 512 = 8.2 → 向上取整
盘块号 = 10000 + 8 = 10008
```

**(2) 分解后检索f4的读盘次数**

```
符号目录大小 = (1000 x 8) / 512 = 15.6 → 16块
最少：1 + 1 = 2次
最多：16 + 1 = 17次
平均：8.5 + 1 = 9.5次（相比分解前47.5次，降至20%）
```

### 4.3 多级索引地址计算

**题目**：盘块4KB，每块放1024个地址，10个直接 + 1次间接 + 1次二级 + 1次三级。

**(1) 从位置9000读1个字节**

```
逻辑块号 = 9000 / 4096 = 2
直接地址 iaddr(2) 直接指向数据块
读盘1次
```

**(2) 从位置180000读1KB**

```
首字节：180000 / 4096 = 43
末字节：181023 / 4096 = 44
逻辑块43、44落在一次间接范围（iaddr(10)）
读盘：索引块1次 + 2个数据块 = 3次
```

**(3) 从位置4200000读100KB**

```
首字节：4200000 / 4096 = 1025
末字节：4302399 / 4096 = 1050
逻辑块1025落在二次间接范围（iaddr(11)）
读盘：二次索引1次 + 一次索引2次 + 26个数据块 = 29次
```

### 4.4 成组链接法实例

**题目**：UNIX系统空闲块成组连接，文件A需5个盘块，文件B删除占用333、334、404、405、782。

**分配过程**：
- 从专用块（栈）中依次弹出空闲块号
- 分配顺序：12、56、49、50、51
- 每分配一个，栈深减1
- 当栈空时，读入下一组盘块号填充栈

**回收过程**：
- 将释放的块号依次压入栈
- 69入栈，栈深+1
- 75入栈，栈深+1
- 87入栈，栈深+1
- 当栈满（100块）时，将当前栈内容写入一个新盘块，该盘块成为新的组首

## 5. 真题/实战

### 5.1 文件类型识别

**场景**：Linux系统中文件颜色含义

| 颜色 | 标记 | 文件类型 |
|------|------|----------|
| 绿色 | | 可执行文件 |
| 红色 | | 压缩文件 |
| 蓝色 | d | 目录文件 |
| 黄色 | c | 字符设备文件 |
| 深黄色 | p | 管道文件 |
| 无特殊色 | - | 普通文件 |

### 5.2 文件属性与目录结构

**三种必要文件属性**：文件名、数据位置（磁盘块号）、访问权限

**文件属性三类**：
1. **一般属性**：文件名、创建者、拥有者、大小、时间戳
2. **数据位置属性**：存储文件内容的磁盘块号
3. **安全属性**：存取权限（读/写/执行/删除）、保护密码

### 5.3 文件搜索过程

**Windows一体化目录搜索**（C:\WINDOWS\system32\drivers\etc\hosts）：

```
C:\(内存) → 读WINDOWS目录块 → 读system32目录块
→ 读drivers目录块 → 读etc目录块 → 获取hosts文件目录项
```

**Linux分解式目录搜索**（/usr/ast/mbox）：

```
根目录(内存) → 读usr目录项(符号) → 读基本目录获取usr的inode
→ 读ast目录项 → 读ast的inode → 读mbox目录项 → 读mbox的inode
```

**优化**：设置当前目录，将常用目录读入内存，减少搜索路径长度。

### 5.4 Linux Ext2文件系统

**Ext2块组结构**（每个块组包含）：
1. **超级块（Superblock）**：文件系统全局信息（inode数、块数、空闲数等）
2. **组描述符表**：各块组的块位图位置、inode位图位置等
3. **块位图**：本组数据块使用情况
4. **inode位图**：inode表使用情况
5. **inode表**：本组所有inode（128字节/个）
6. **数据块**：实际文件内容

**Ext2目录项结构**：
```c
struct ext2_dir_entry_2 {
    _u32 inode;       // inode号
    _u16 rec_len;     // 目录项长度（可变）
    _u8 name_len;     // 文件名长度（最长256B）
    _u8 file_type;    // 文件类型
    char name[256];   // 文件名
};
```

### 5.5 VFS与具体文件系统关系

```
应用层：标准系统调用 open(), read(), write(), close()
    ↓
虚拟层：VFS → sys_open(), sys_read(), sys_write(), sys_close()
    ↓
实现层：Minix / Ext2 / FAT / ... （通过 f_op 指针调用具体实现）
```

### 5.6 RAID磁盘阵列

| RAID级别 | 特点 | 利用率 | 可靠性 |
|----------|------|--------|--------|
| RAID 0 | 仅并行交叉存取，无冗余 | 100% | 低 |
| RAID 1 | 磁盘镜像 | 50% | 高 |
| RAID 3 | 并行传输 + 专用校验盘 | (n-1)/n | 较高 |
| RAID 5 | 独立传送 + 校验信息散布 | (n-1)/n | 高 |
| RAID 6/7 | 强化版，专用异步校验盘 | 更低 | 最高 |

RAID 5 磁盘利用率 = (n-1)/n（如8块盘 → 87.5%）

## 6. 巩固练习

### 练习1：FAT文件系统容量计算

**题目**：磁盘容量256MB，簇大小4KB，FAT有4个副本，FAT16格式。

**解答**：
- 簇数 = 256MB / 4KB = 64K
- 每项16位 = 2字节
- FAT占用 = 2B x 64K x 4 = 512KB
- 开销 ≈ 0.2%

### 练习2：FAT32表开销

**题目**：硬盘500MB，簇1KB，4个副本，FAT32格式。

**解答**：
- 簇数 = 500MB / 1KB = 500K
- 每项32位 = 4字节
- FAT占用 = 4B x 500K x 4 = 8MB
- 开销 ≈ 1.3%

### 练习3：位示图分配

**题目**：位示图管理空闲盘块，盘块1KB，分配两个盘块。

**解答**：
- 从位示图中找两个值为0的位
- 计算对应盘块号：n = 字大小 x i + j
- 将位标记为1（已分配）

### 练习4：混合索引最大文件

**题目**：盘块1KB，指针4B，直接地址6项，一级索引2项，二级索引1项。

**解答**：
- 每块地址数 = 1KB / 4B = 256
- 直接地址：6块
- 一级索引：2 x 256 = 512块
- 二级索引：256 x 256 = 65536块
- 最大文件 = (6 + 512 + 65536) x 1KB ≈ **64MB**

### 练习5：inode指针空间计算

**题目**：inode 128B，指针4B，状态信息68B，块大小8KB。

**解答**：
- 指针空间 = 128 - 68 = 60B
- 指针数 = 60 / 4 = 15个
- 假设分配：10直接 + 1一次间接 + 1二次间接 + 1三次间接 + 2保留
- 直接：10 x 8KB = 80KB
- 一次间接：1 x (8KB/4B) x 8KB = 1024 x 8KB = 8MB
- 二次间接：1 x 1024 x 1024 x 8KB = 8GB
- 三次间接：1 x 1024^3 x 8KB = 8TB

### 练习6：链接数一致性检查

**题目**：检查UNIX文件系统中共享文件的链接计数一致性。

**解答**：
- 遍历所有目录，统计每个inode号出现的次数
- 与inode中的count值比较
- count > 计数值：用正确计数值重新赋值
- count < 计数值：将count置为正确值

## 7. 总结复盘

### 易错点

1. **逻辑地址 vs 物理地址**：逻辑地址从0开始按字节/块编号，物理地址是磁盘块号。文件内容最小单位是字节，磁盘分配最小单位是块。

2. **目录分解后仍需2次读盘**：符号目录检索（可能多次）+ 基本目录读取（1次），不能忘记基本目录那1次。

3. **多级索引读盘次数**：需要逐级读取索引块，不是直接跳到数据块。如二次间接 = 读二级索引块 + 读一级索引块 + 读数据块。

4. **FAT表双份**：FAT12/FAT16/FAT32都有两张相同FAT表（FAT1和FAT2），计算开销时需乘以副本数。

5. **成组链接法栈满/栈空处理**：
   - 分配时栈空：读入下一组盘块号填充栈，继续分配
   - 回收时栈满：将当前栈写入新盘块，该盘块成为组首，重新开始

6. **位示图行列计算**：注意字大小（如16位或32位），n = 字大小 x i + j，不要混淆。

### 关键要点

1. **文件逻辑结构三分类**：流式文件（字节串）、记录式文件（定长/不定长记录）、复杂文件（魔数标识）。

2. **三种外存组织**：连续（顺序访问快）、链接（FAT解决随机访问）、索引（直接访问 + 多级索引支持大文件）。

3. **目录分解（Linux inode）**：将文件名与属性分离，平均读盘次数降至原来的20%，是现代OS的标准做法。

4. **空闲空间管理**：位示图法直观适合中等规模，成组链接法是UNIX/Linux标准。

5. **磁盘高速缓存**：在内存中缓存磁盘块副本，采用LRU等置换算法，UNIX用SYNC周期性写回。

6. **磁盘容错三级**：SFT-I（双份目录/FAT + 热修复）、SFT-II（磁盘镜像/双工）、集群（热备份/互为备份/公用磁盘）。

7. **数据一致性**：检查重复文件一致性和链接数一致性（inode count vs 目录中的实际引用数）。

8. **事务原子性**：要么全部完成（redo），要么全部回滚（undo），检查点机制减少恢复开销。
