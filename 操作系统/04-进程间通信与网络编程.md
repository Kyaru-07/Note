# 第7讲 进程间通信

## 1. 概念引入

一般情况下，进程是一个封闭实体，分配给不同进程的存储器不重合、不交错，每个进程只能通过系统调用函数与操作系统交互，进程之间相对独立，互不影响。但为了合作完成一个任务，很多进程之间需要进行通信、进行数据传输，甚至还需对相关活动进行协调。

由于同一计算机系统上多进程本来就共享所有系统资源，所以在进程间进行资源共享、数据共享是顺理成章的事情。但进程间通信必须借助操作系统提供的通信机制来实现。

**System V IPC** 源自于 System V Unix，是一种进程间通信的设施（Interprocess Communication），包括以下三种设施：
- 消息队列（Message Queue）
- 共享内存（Shared Memory）
- 信号量（Semaphore）

这三种设施与管道（Pipe）机制一起，构成 Linux 进程间高级通信机制，以支持进程间大量数据的传送，适应不同应用需要。

### 本讲学习目标

1. 理解利用 PIPE 和 FIFO 实现进程间数据通信的原理，掌握相应编程方法，理解 Linux 管道命令实现原理
2. 理解消息队列结构和通信原理，掌握利用消息队列进行进程间数据通信的编程方法
3. 理解共享内存原理，掌握利用共享内存实现进程间数据共享的编程方法
4. 理解 IPC 信号量 API 函数使用方法，掌握利用 IPC 信号量实现进程同步互斥编程方法

### 主要内容

- 管道通信：无名管道（PIPE）、命名管道（FIFO）
- 消息队列（Message Queue）
- 共享内存（Shared Memory）
- IPC 信号量（InterProcess Communication Semaphore）

## 2. 核心概念：管道通信（Pipe）

### 2.1 什么是管道

**定义：** 管道是一种具有生活中"输油管"、"输水管"特性的进程间通信机制。一个进程从管道一端写入数据，另一个进程从管道另一端读出，从而实现了数据传输。

**实质：** 管道是操作系统内核实现的一个内存缓冲区。

**典型应用：** 多个命令的输入输出首尾相接

```bash
$ cmd1 | cmd2 | ...
$ more /etc/passwd | sort
$ find . -name "*.c~" | grep p5 | rm
```

### 2.2 命名管道 FIFO（Named Pipe）

**什么是命名管道：**
- 一种特殊的文件类型，称为 FIFO，文件类型为 `p`
- 具有文件名、创建时间、访问权限等几乎所有的文件属性外观
- 数据存储在操作系统内核的缓冲区内存中，不写入磁盘块，因此管道文件在磁盘上没有数据块
- 操作方法：文件操作命令（如 `echo`）或 UNIX I/O 函数（`read`/`write`）
- 与普通文件相比：数据不经过外存，读写速度快

**创建 FIFO 文件：**

```bash
# 命令创建
$ mkfifo /tmp/myfifo1
$ ls -lF /tmp/myfifo1
```

```c
// 编程创建
#include <sys/types.h>
#include <sys/stat.h>
int mkfifo(const char *filename, mode_t mode);
```

**示例（fifo1.c）：** 在 `/tmp` 目录下创建一个名为 `myfifo` 的 FIFO 文件

```c
#include "wrapper.h"
int main()
{
    int res = mkfifo("/tmp/myfifo", 0777);
    if (res == 0) printf("FIFO created\n");
    exit(EXIT_SUCCESS);
}
```

运行结果：
```
$ ./myfifo
FIFO created
$ ls -lF /tmp/myfifo
prwxr-xr-x 1 can can 0 2015-12-26 20:00 /tmp/myfifo|
```

### 2.3 使用命令访问 FIFO

直接把 FIFO 当文件操作，使用 `echo` 命令写入数据：

```bash
$ echo "Hello FIFO" > /tmp/myfifo
$ cat /tmp/myfifo
Hello FIFO
```

### 2.4 不同程序使用 FIFO 通信

**示例：** 利用 `/tmp/myfifo` 实现进程间通信，`fifowrite.c` 将命令行参数 `argv[1]` 的值写入 `/tmp/myfifo`，`fiforead.c` 从 `/tmp/myfifo` 读出数据并显示

**fifowrite.c（写端）：**

```c
#include "wrapper.h"
int main(int argc, char *argv[])
{
    int pipe_fd;
    if (argc != 2) {
        printf("need a string\n");
        exit(1);
    }
    printf("Process %d opening FIFO O_WRONLY\n", getpid());
    pipe_fd = Open("/tmp/myfifo", O_WRONLY, 0);
    printf("Process %d result %d\n", getpid(), pipe_fd);
    Write(pipe_fd, argv[1], strlen(argv[1]) + 1);
    Close(pipe_fd);
    return 0;
}
```

**fiforead.c（读端）：**

```c
#include "wrapper.h"
int main(int argc, char *argv[])
{
    int pipe_fd;
    char info[128] = {0};
    printf("Process %d opening FIFO O_RDONLY\n", getpid());
    pipe_fd = Open("/tmp/myfifo", O_RDONLY, 0);
    printf("Process %d result %d\n", getpid(), pipe_fd);
    Read(pipe_fd, info, sizeof(info));
    printf("%s\n", info);
    Close(pipe_fd);
    return 0;
}
```

编译运行：
```bash
$ gcc -o fifowrite fifowrite.c -L. -lcasapp
$ gcc -o fiforead fiforead.c -L. -lcasapp
```

### 2.5 利用 FIFO 传输任意类型数据

任何数据结构都存在于某个内存块中，将该内存块看成 `write`、`read` 函数调用的缓冲区，就可通过 FIFO 将任何数据结构传递给其它进程。

**方法：**
- 假设要传递类型为 `T` 的变量 `var`，将其地址 `&var` 和长度 `sizeof(T)` 作为缓冲区
- 发送方将缓冲区内容写入管道
- 接收方将收到信息存入缓冲区，并将缓冲区看成一个 `T` 类型指针

对于数组 `T arr[N]`，可以每次读写一个数组元素，或将整个数组作为一个整体进行发送。

## 3. 核心概念：无名管道（PIPE）

### 3.1 什么是无名管道

由于子进程可以直接继承父进程拥有的打开文件描述符资源，在父子进程间采用管道机制通信，甚至连文件名都可以省略，称为**无名管道**。

### 3.2 无名管道创建

```c
#include <unistd.h>
int pipe(int fds[2]);
```

- 返回文件描述符 `fds[1]` 是管道的写端
- `fds[0]` 是管道的读端

### 3.3 示例1：验证管道工作（pipe1.c）

```c
#include "wrapper.h"
int main()
{
    int count;
    int fds[2];
    const char some_data[] = "1234567890";
    char buffer[BUFSIZ + 1];
    memset(buffer, '\0', sizeof(buffer));

    pipe(fds);
    count = Write(fds[1], (void *)some_data, strlen(some_data));
    printf("Wrote %d bytes\n", count);

    count = Read(fds[0], (void *)buffer, BUFSIZ);
    printf("Read %d bytes: %s\n", count, buffer);
    exit(EXIT_SUCCESS);
}
```

运行结果：
```
$ gcc pipe1.c -o pipe1 -L. -lwrapper
$ ./pipe1
Wrote 10 bytes
Read 10 bytes: 1234567890
```

### 3.4 示例2：父进程通过管道传递结构体给子进程（pipe2.c）

创建一个子进程，父进程通过无名管道将一个学生信息结构体传递给子进程输出显示。结构体包括学号、姓名、语文、数学、英语五个字段。

```c
#include "wrapper.h"
struct grade {
    char sno[20];       // 学号
    char name[10];      // 姓名
    float chinese;      // 语文
    float math;         // 数学
    float english;      // 英语
};
int main()
{
    int count;
    int fds[2];
    char buffer[BUFSIZ + 1];
    pid_t pid;
    struct grade grad = {"201441401123", "李向阳", 89.1, 70, 80};
    struct grade *p = buf;

    pipe(fds);
    pid = Fork();
    if (pid == 0) {          // 子进程
        Close(fds[1]);
        count = Read(fds[0], buf, BUFSIZ);
        printf("Read %d bytes:\n学号=%s\n姓名=%s\n语文=%3.1f\n数学=%3.1f\n英语=%3.1f\n",
               count, p->sno, p->name, p->chinese, p->math, p->english);
        Close(fds[0]);
        exit(EXIT_SUCCESS);
    } else {                 // 父进程
        Close(fds[0]);
        count = Write(fds[1], (void *)&grad, sizeof(struct grade));
        printf("Wrote %d bytes\n", count);
        Close(fds[1]);
        exit(EXIT_SUCCESS);
    }
}
```

运行结果：
```
$ gcc pipe2.c -o pipe2 -L. -lwrapper
$ ./pipe2
Wrote 44 bytes
Read 44 bytes:
学号=201441401123
姓名=李向阳
语文=89.1
数学=70.0
英语=80.0
```

### 3.5 使用 PIPE 实现管道命令（pipe3.c）

子进程通过管道连接，父进程将字符串 `"1234567890"` 传递给子进程处理，子进程执行 `od` 程序对收到的信息进行计数处理。

**思路：** 先创建管道，创建子进程，通过 `dup` 调用将父进程输出导入到子进程输入

```c
#include "wrapper.h"
int main()
{
    int count, pid;
    int fds[2];
    const char some_data[] = "1234567890";
    char buffer[BUFSIZ + 1];
    memset(buffer, '\0', sizeof(buffer));
    pipe(fds);
    pid = Fork();
    if (pid == 0) {
        Close(0);
        dup(fds[0]);
        Close(fds[0]);
        Close(fds[1]);
        execlp("od", "od", "-c", (char *)0);
        exit(EXIT_FAILURE);
    } else {
        Close(fds[0]);
        count = Write(fds[1], some_data, strlen(some_data));
        Close(fds[1]);
        printf("Wrote %d bytes\n", count);
        exit(EXIT_SUCCESS);
    }
}
```

运行结果：
```
$ gcc pipe3.c -o pipe3 -L. -lwrapper
$ ./pipe3
Wrote 10 bytes
0000000   1   2   3   4   5   6   7   8   9   0
0000012
```

### 3.6 使用 FIFO 的客户/服务器应用程序

**要求：** 通过命名管道编写一个简单的客户/服务器应用程序，客户端发送数据，服务器进程接受请求，将字母转换为大写，处理后返回给客户端。

**设计思路：**
1. **管道设置：**
   - Client -> Server：共用 1 个管道 `serv_fifo`，用 PID 标识来自哪个 Client
   - Server -> Client：每个 Client 一个管道，名称为 `cli_<PID>_fifo`

2. **信息发送和接收缓冲区结构：**

```c
#define BUFFER_SIZE 20
struct data_to_pass_st {
    pid_t client_pid;                       // 客户进程的 PID
    char  some_data[BUFFER_SIZE - 1];       // 存放消息的用户缓冲区
};
```

3. **Server 设计：**

```c
// (1) 打开公共管道，接收数据
mkfifo(SERVER_FIFO_NAME, 0777);
server_fifo_fd = Open(SERVER_FIFO_NAME, O_RDONLY);
sleep(10);              /* 客户进程先睡眠 10 秒钟 */
do {
    read_res = Read(server_fifo_fd, &my_data, sizeof(my_data));

    // (2) 数据转换
    tmp_char_ptr = my_data.some_data;
    while (*tmp_char_ptr) {
        *tmp_char_ptr = toupper(*tmp_char_ptr);
        tmp_char_ptr++;
    }

    // (3) 打开客户管道
    sprintf(client_fifo, CLIENT_FIFO_NAME, my_data.client_pid);
    client_fifo_fd = Open(client_fifo, O_WRONLY);
    Write(client_fifo_fd, &my_data, sizeof(my_data));
    Close(client_fifo_fd);
} while (read_res > 0);

Close(server_fifo_fd);
unlink(SERVER_FIFO_NAME);
exit(EXIT_SUCCESS);
```

4. **Client 设计：**

```c
// (1) 创建客户 FIFO，准备信息
server_fifo_fd = Open(SERVER_FIFO_NAME, O_WRONLY);
my_data.client_pid = getpid();
sprintf(client_fifo, CLIENT_FIFO_NAME, my_data.client_pid);
mkfifo(client_fifo, 0777);

// (2) 循环发送数据
for (i = 0; i < 5; i++) {
    sprintf(my_data.some_data, "Hello from %d", my_data.client_pid);
    printf("%d sent %s, ", my_data.client_pid, my_data.some_data);
    write(server_fifo_fd, &my_data, sizeof(my_data));

    // (3) 读回数据
    client_fifo_fd = open(client_fifo, O_RDONLY);
    Read(client_fifo_fd, &my_data, sizeof(my_data));
    printf("received: %s\n", my_data.some_data);
    close(client_fifo_fd);
}

// (4) 善后处理
close(server_fifo_fd);
unlink(client_fifo);
exit(EXIT_SUCCESS);
```

5. **运行测试：**

```bash
$ ./server &
$ for i in 1 2 3 4 5; do ./client & done
531 sent Hello from 531, received: HELLO FROM 531
532 sent Hello from 532, received: HELLO FROM 532
```

## 4. 核心概念：消息队列（Message Queue）

### 4.1 基本思想

进程间以数据块（消息）为单位来传递数据，是进程之间传递数据的简单有效方法。它独立于发送和接收进程而存在，消除了在同步命名管道的打开和关闭时遇到的麻烦。

**限制：**
- `MSGMAX`：消息的最大长度
- `MSGMNB`：队列的最大长度

每个数据块可被看成某种类型，可发送任何类型数据。

### 4.2 消息队列结构

**消息类型结构体：**

```c
struct msgbuf {
    long mtype;                 /* 消息的类型，必须为正数 */
    char mtext[BUFFER_SIZE];    /* 消息正文 */
};
```

### 4.3 消息队列通信实例

**消息队列创建（msgcreate.c）：** 创建键值为 `0x12345` 的消息队列

```c
#include "wrapper.h"
int main(int argc, char *argv[])
{
    int rtn;
    int msqid;
    key_t key;
    if (argc <= 1) {
        fprintf(stderr, "请以./msgcreate <key>形式运行给出消息队列键值\n");
        exit(2);
    }
    sscanf(argv[1], "%x", &key);    // 将参数转换成十六进制数
    msqid = Msgget(key, IPC_CREAT | 0644);
    exit(0);
}
```

### 4.4 通过消息队列传输任意类型的数据

要传送类型为 `T` 的变量 `var`，消息缓冲区是地址为 `buf` 的缓冲区：
1. 将变量 `var` 所在单元看成一个缓冲区
2. 将缓冲区内容写入消息队列
3. 接收方将收到信息存入缓冲区，并将缓冲区看成一个 `T` 类型指针

## 5. 核心概念：共享内存（Shared Memory）

### 5.1 基于共享内存进程通信的基本原理

根据进程请求分配一块合适大小的存储块，各请求进程在其地址空间为该共享内存块分配程序地址，将程序地址赋值给变量指针，然后通过变量指针读写共享内存单元，从而实现数据传输。

**关键区别：**
- 一般情况下，OS 分配给进程的物理内存块都是独享的，不同进程的虚拟地址对应不同的物理内存块
- 只有共享内存是两个进程共有的内存块，两个进程中对应共享内存的地址区域对应着相同的物理内存块
- 各进程分配给共享内存块的逻辑地址范围可能不相同

### 5.2 共享内存相关 API 函数

**创建共享内存：**

```c
int shmget(key_t key, size_t size, int flag);
// 示例：shmget(key, 4096, IPC_CREAT | 0644);
```

**将共享内存映射到进程的逻辑地址空间：**

```c
void *shmat(int shm_id, const void *shm_addr, int flag);
// 示例：shmptr = shmat(shmid, 0, 0);
```

**取消映射：**

```c
int shmdt(char *shmaddr);
```

**销毁共享内存：**

```c
Shmctl(shmid, IPC_RMID, NULL);
```

### 5.4 共享内存通信实例：生产者/消费者

一个生产者/消费者应用，生产者进程 `shmprod.c` 循环读取用户从键盘输入的信息，通过共享内存传递给消费者进程 `shmcons.c` 显示输出，用户输入 `"end"` 结束程序。

**共享内存块结构和同步设计：**

```c
#define TEXT_SZ 2048
struct shared_mm {
    int  flag;
    char buf[TEXT_SZ];
};
```

标志单元 `flag` 的含义：
- `flag = 0`：空闲内存为空（初始状态）
- `flag = 1`：共享内存有新数据
- `flag = 2`：数据传输结束

**同步机制：**
- 生产者 `shmprod.c` 仅在 `flag=0` 时才能将数据写入共享内存，并将 `flag` 修改成 1；若是写入结束串 `"end"` 则将 `flag` 改成 2
- 消费者 `shmcons.c` 仅当 `flag=1` 时才能从共享内存读出数据，并将 `flag` 标志修改为 0

**生产者进程（shmprod.c）：**

```c
int main(int argc, char *argv[])
{
    int rtn;
    int shmid;
    key_t key;
    if (argc <= 1) {
        fprintf(stderr, "请以./msgcreate <key>形式运行给出消息队列键值\n");
        exit(2);
    }
    sscanf(argv[1], "%x", &key);
    shmid = Shmget(key, sizeof(struct shared_mm), IPC_CREAT | 0644);
    shmptr = (struct shared_mm *)Shmat(shmid, 0, 0);
    for (;;) {
        while (shmptr->flag != 0);      // 共享内存空闲则退出
        scanf("%s", shmptr->buf);        // 从标准输入读入信息，并写入共享内存
        if (strcmp(shmptr->buf, "end") == 0) {
            shmptr->flag = 2;
            break;
        } else {
            shmptr->flag = 1;
        }
    }
    Shmdt(shmptr);
    exit(0);
}
```

**消费者进程（shmcons.c）：**

```c
int main(int argc, char *argv[])
{
    int rtn;
    int shmid;
    key_t key;
    if (argc <= 1) {
        fprintf(stderr, "请以./msgcreate <key>形式运行给出消息队列键值\n");
        exit(2);
    }
    sscanf(argv[1], "%x", &key);
    shmid = Shmget(key, sizeof(struct shared_mm), IPC_CREAT | 0644);
    shmptr = (struct shared_mm *)Shmat(shmid, 0, 0);
    for (;;) {
        while (shmptr->flag == 0);      // 共享内存有新数据则退出
        printf("%s\n", shmptr->buf);     // 从共享内存读信息并输出
        shmptr->flag = 0;
        if (strcmp(shmptr->buf, "end") == 0)
            break;
    }
    Shmdt(shmptr);
    exit(0);
}
```

## 6. 核心概念：IPC 信号量（Semaphore）

### 6.1 信号量集结构

**信号量集描述（struct semid_ds）：**

```c
struct semid_ds {
    struct ipc_perm sem_perm;           /* 该信号量集的操作权限 */
    struct sem *sem_base;               /* 该数组的元素是信号量结构 */
    ushort sem_nsems;                   /* sem_base 数组的个数 */
    struct sem_queue *sem_pending;      /* 被挂起进程队列 */
    ...
};
```

**每个信号量描述（struct sem）：**

```c
struct sem {
    int     semval;     /* 信号量当前值 */
    int     sempid;     /* 最后操作的进程 PID */
    ushort  semcnt;     /* 等待信号量大于当前值的进程数 */
    ushort  semzcnt;    /* 等待信号量等于 0 的进程数 */
};
```

### 6.2 信号量基本操作函数

**获取或创建信号量集：**

```c
int semget(key_t key, int nsems, int flag);
// nsems：信号量集中信号量的个数
// flag：操作标志，如 IPC_CREAT
```

**信号值增加或减少：**

```c
int semop(int sem_id, struct sembuf *sops, unsigned int nsops);
```

参数说明：
- `semid`：是 `semget` 函数返回的信号量描述符
- `nsops`：是本次操作的信号量数目，也是 `sops` 指向的数组大小
- `sops`：指向一个类型为 `sembuf` 的结构体数组

```c
struct sembuf {
    ushort sem_num;     /* 待操作的信号量在信号量集的索引值 */
    short  sem_op;      /* 信号量操作（正数、负数或 0） */
    short  sem_flg;     /* 操作标志，为 IPC_NOWAIT 或 SEM_UNDO */
};
```

**信号量控制（如赋初值）：**

```c
int semctl(int semid, int semnum, int cmd, union semun arg);
```

参数说明：
- `semid`：信号量集描述符
- `semnum`：待操作信号在信号量集 `semid` 的索引
- `cmd`：指定具体操作类型
- `arg`：对应的参数

常用操作类型：
- `SETVAL`：设置 `semnum` 所代表信号量的值为 `arg.val`
- `SETALL`：通过 `arg.val` 更新所有信号量值
- `IPC_RMID`：从内核主存中删除信号量集
- `GETVAL`：返回 `semnum` 所代表信号量的值

**联合体 union semun 类型定义：**

```c
union semun {
    int val;
    struct semid_ds *buf;
    ...
};
```

### 6.3 自定义 P、V 操作库函数

**创建信号量：**

```c
int createsem(key_t key) {
    return semget(key, 1, IPC_CREAT | 0666);
}
```

**获取信号量：**

```c
int getsem(key_t key) {
    return semget(key, 1, 0666);
}
```

**初始化信号量：**

```c
int initsem(int semid, int initval) {
    union semun arg;
    arg.val = initval;
    return semctl(semid, 1, SETVAL, arg);
}
```

**删除信号量：**

```c
int delsem(int semid) {
    union semun arg;
    semctl(semid, 0, IPC_RMID, arg);
}
```

**P 操作（信号量减 1）：**

```c
int P(int semid) {
    struct sembuf operation;
    operation.sem_num = 0;
    operation.sem_op = -1;
    return semop(semid, &operation, 1);
}
```

**V 操作（信号量加 1）：**

```c
int V(int semid) {
    struct sembuf operation;
    operation.sem_num = 0;
    operation.sem_op = 1;
    return semop(semid, &operation, 1);
}
```

## 7. 总结复盘

### 关键术语回顾

| 术语 | 英文 | 说明 |
|------|------|------|
| 管道 | Pipe | 进程间通信的内存缓冲区机制 |
| 无名管道 | PIPE | 父子进程间使用的管道，无文件名 |
| 命名管道 | FIFO | 有文件名的管道，任意进程可访问 |
| 消息队列 | Message Queue | 以数据块为单位的进程间通信 |
| 共享内存 | Shared Memory | 多进程共享同一块物理内存 |
| 信号量 | Semaphore | 用于进程间同步互斥的计数器 |

### 四种 IPC 机制对比

| 机制 | 特点 | 适用场景 |
|------|------|----------|
| 管道（PIPE/FIFO） | 简单，基于文件 I/O，单向通信 | 父子进程间简单数据传输 |
| 消息队列 | 独立于进程，支持多种数据类型 | 不相关进程间的结构化数据通信 |
| 共享内存 | 最快的 IPC，直接读写内存 | 大量数据的高速共享 |
| 信号量 | 用于同步和互斥控制 | 配合其他 IPC 机制使用 |

### 重点知识

1. **管道**是内核中的内存缓冲区，数据不经过磁盘，读写速度快
2. **无名管道**通过 `pipe()` 创建，只能用于父子进程间通信
3. **命名管道**通过 `mkfifo()` 创建，可用于任意进程间通信
4. **共享内存**是最快的 IPC 方式，但需要配合信号量解决同步问题
5. **IPC 信号量**的 P 操作（`sem_op=-1`）和 V 操作（`sem_op=1`）是实现进程同步的核心

## 2. 核心概念：客户端/服务器编程模型

客户端-服务器模型中的基本操作是**事务（Transaction）**：

1. **第一步：** 客户端向服务器发送一个请求，发起一个事务

3. **第三步：** 服务器给客户端发送一个响应，并等待下一个请求

## 3. 核心概念：网络通信结构和 Internet 连接

### 3.1 网络通信的定义

网络通信是通过计算机网络进行信息传递和数据交流的过程，通过各种协议和技术实现数据的传输和接收，网络通信实际上是**跨越网络的进程间通信**。

### 3.2 TCP/IP 协议

**TCP/IP（Transmission Control Protocol/Internet Protocol，传输控制协议/网际协议）** 是指能够在多个不同网络间实现信息传输的协议簇。几乎所有的计算机系统都内置该协议的支持。

TCP（传输控制协议）和 IP（网际协议）一起工作来保证数据在计算机网络上的可靠传输。TCP/IP 协议不仅仅指的是 TCP 和 IP 两个协议，而是指一个由 FTP、SMTP、TCP、UDP、IP 等协议构成的协议簇，只是因为 TCP/IP 协议中 TCP 协议和 IP 协议最具代表性，所以被称为 TCP/IP 协议。

### 3.3 IPC 机制 vs Socket 网络通信

**IPC 机制（消息队列、共享内存、IPC 信号量、管道）：**
- 结构麻烦、不易使用、仅用于同机进程间通信
- 仅用于 Linux/Unix 环境

**Socket（套接字）网络通信：**
- 标准、规范，基于 TCP/IP 协议
- 两种模式：
  - TCP 通信（与管道类似）
  - UDP 通信（与消息队列类似）
- 易于理解、使用，使用广泛
- 既可用于单机进程间通信，也可用于不同计算机进程间通信

### 3.4 网络通信结构

分层设计：各层分工明确，可靠性好，性能高。

### 3.5 套接字编程模型

- 结构：网卡、TCP 协议、套接字（Socket）
- 类比：
  - 网卡 = 单位门牌号
  - TCP/IP 协议 = 收发室
  - 套接字 = 信箱号
- 套接字：含有进程接收信息的完整地址（Socket 地址 = IP 地址 + 端口号）

### 3.6 TCP 连接

TCP 连接比喻为连接通信双方套接字的一条通信线路：
- 通信前建立，通信结束拆除
- 一条 TCP 连接实际上就是一个文件描述符
- 可用 `read`/`write` 或 `send`/`recv` 进行数据收发
- 地址格式：`(cliaddr:cliport, servaddr:servport)`

### 3.7 因特网连接实例（网页浏览）

- 服务器端口号：规定为 80
- 客户端端口号：随机分配（如 12345）

## 4. 核心概念：套接字地址与设置方法

### 4.1 套接字地址结构

**早期定义（16 字节，适用于各种网络环境）：**

```c
struct sockaddr {
    unsigned short sa_family;    /* protocol family */
    char sa_data[14];
};
```

**适合 Internet 环境的套接字地址结构：**

```c
/* Internet-style socket address structure */
struct sockaddr_in {
    unsigned short sin_family;    /* 协议族，网络序 */
    unsigned short sin_port;      /* 端口号，2 字节，网络序 */
    struct in_addr sin_addr;      /* IP 地址，4 字节，网络序 */
    unsigned char sin_zero[8];    /* 填充，必须为 0，8 字节 */
};

struct in_addr {
    unsigned int s_addr;  /* network byte order (big-endian) */
};
```

### 4.2 字节序（Byte Order）

**问题的提出：**

整数在内存中的存储顺序：
- 字符串（`char *X="ABCDEFGHIJ"`）：按地址顺序排列
- 整数：`unsigned int B[2]={0x12345678, 0xabcdef}`，在内存中的排列取决于字节序

**大端模式（Big-Endian）：** 高位在低地址字节，一般为网络序

```
A     A+4    &T    K
12    34     56    78     00    cd    ef    ab     12    D2    34    56    78
```

**小端模式（Little-Endian）：** 低位在低地址字节，一般为主机序

```
A     A+4    &T    K
78    56     34    12     ef    ab    00    cd     78    04    56    34    12    04    D2
```

### 4.3 IP 地址和字节序

32 位整数（大端模式，网络序）：

```c
struct in_addr IP1 = htonl(0x8002c2f2);
// 128=0x80, 2=0x02, 193=0xc2, 242=0xf2
// 点分十进制：128.2.193.242
```

**转换函数：**

```c
int inet_aton(const char *cp, struct in_addr *inp);  // 字符串 -> 32 位整数
char *inet_ntoa(struct in_addr in);                    // 32 位整数 -> 字符串
```

**域名结构体（struct hostent）：**

**open_client_sock 包装函数：**

```c
int open_client_sock(char *hostname, int port)
{
    int client_sock;
    struct hostent *hp;
    struct sockaddr_in serveraddr;

    if ((client_sock = socket(AF_INET, SOCK_STREAM, 0)) < 0)
        return -1;        /* if error return -1 */

    /* Fill in the server's IP address and port */
    if ((hp = gethostbyname(hostname)) == NULL)
        return -2;       /* if error return -2 */
    bzero((char *)&serveraddr, sizeof(serveraddr));
    serveraddr.sin_family = AF_INET;
    bcopy((char *)hp->h_addr_list[0],
          (char *)&serveraddr.sin_addr.s_addr, hp->h_length);
    serveraddr.sin_port = htons(port);

    /* Establish a connection with the server */
    if (connect(client_sock, (SA *)&serveraddr, sizeof(serveraddr)) < 0)
        return -1;
    return client_sock;
}
```

void toggle(int conn_sock)
{
    size_t n;
    int i;
    char buf[MAXLINE];

    while ((n = recv(conn_sock, buf, MAXLINE, 0)) > 0) {
        printf("toggle server received %d bytes\n", n);
        for (i = 0; i < n; i++)
            if (isupper(buf[i]))
                buf[i] = tolower(buf[i]);
            else if (islower(buf[i]))
                buf[i] = toupper(buf[i]);
        send(conn_sock, buf, n, 0);
    }
}

### 7.1 为何要用 Web 编程

**前述网络编程方法的问题：**
- 客户与服务器间传送数据的含义由双方预先约定
- 应用升级要重新修改和部署服务器与客户端
- 一般客户端分发和升级比较麻烦

**Web 编程优点：**
- 只需开发服务器端，客户端为浏览器
- 如果升级只需直接升级服务器端
- 服务器端与客户端之间有规范的通信协议

### 7.5 小型 Web 服务器 weblet

#### 功能

1. **请求行处理：**

#### 处理 HTTP 事务（process_trans 函数）

```c
void process_trans(int fd)  // fd 为网络连接
{
    int static_flag;
    struct stat sbuf;
    char buf[MAXLINE], method[MAXLINE], uri[MAXLINE], version[MAXLINE];
    char filename[MAXLINE], cgiargs[MAXLINE];
    rio_t rio;

    /* read request line and headers */
    rio_readinitb(&rio, fd);
    rio_readlineb(&rio, buf, MAXLINE);
    sscanf(buf, "%s %s %s", method, uri, version);
    if (strcasecmp(method, "GET")) {
        error_request(fd, method, "501", "Not Implemented",
                "weblet does not implement this method");
        return;
    }
    read_requesthdrs(&rio);
}
```

void parse_static_uri(char *uri, char *filename)
{

    if (uri[strlen(uri) - 1] == '/')
        strcat(filename, "home.html");
}

**处理动态网页路径（parse_dynamic_uri 函数）：**

```c
void parse_dynamic_uri(char *uri, char *filename, char *cgiargs)
{
    char *ptr;
    ptr = index(uri, '?');
    if (ptr) {
        strcpy(cgiargs, ptr + 1);
        *ptr = '\0';
    } else
        strcpy(cgiargs, "");
    strcpy(filename, ".");
    strcat(filename, uri);
}
```

- 动态路径的 URI 中必有 `?` 字符
- 以 `uri="/cgi-bin/add?2017&523808"` 为例：
  - `cgiargs = "2017&523808"`
  - `filename = "./cgi-bin/add"`

#### 返回静态网页（feed_static 函数）

```c
void feed_static(int fd, char *filename, int filesize)
{
    int srcfd;
    char *srcp, filetype[MAXLINE], buf[MAXBUF];

    /* Send response headers to client */
    get_filetype(filename, filetype);
    sprintf(buf, "HTTP/1.1 200 OK\r\n");
    sprintf(buf, "%sServer: weblet Web Server\r\n", buf);
    sprintf(buf, "%sContent-length: %d\r\n", buf, filesize);
    sprintf(buf, "%sContent-type: %s\r\n\r\n", buf, filetype);
    rio_writen(fd, buf, strlen(buf));

    /* Send response body to client */
    srcfd = open(filename, O_RDONLY, 0);
    srcp = mmap(0, filesize, PROT_READ, MAP_PRIVATE, srcfd, 0);
    close(srcfd);
    rio_writen(fd, srcp, filesize);
    munmap(srcp, filesize);
}

void get_filetype(char *filename, char *filetype)
{
    if (strstr(filename, ".html"))
        strcpy(filetype, "text/html");
    else if (strstr(filename, ".jpg"))
        strcpy(filetype, "image/jpeg");
    else if (strstr(filename, ".mpeg"))
        strcpy(filetype, "video/mpeg");
    else
        strcpy(filetype, "text/html");
}
```

#### 返回动态网页（feed_dynamic 函数）

```c
void feed_dynamic(int fd, char *filename, char *cgiargs)
{
    char buf[MAXLINE], *emptylist[] = {NULL};
    int pfd[2];

    /* Return first part of HTTP response */
    sprintf(buf, "HTTP/1.0 200 OK\r\n");
    rio_writen(fd, buf, strlen(buf));
    sprintf(buf, "Server: weblet Web Server\r\n");
    rio_writen(fd, buf, strlen(buf));

    pipe(pfd);
    if (fork() == 0) {
        close(pfd[1]);
        dup2(pfd[0], STDIN_FILENO);
        dup2(fd, STDOUT_FILENO);               /* fd 为网络连接 */
        execve(filename, emptylist, environ);   /* 加载运行 CGI 程序 */
    }
    close(pfd[0]);
    write(pfd[1], cgiargs, strlen(cgiargs) + 1);  // 通过管道把参数传递给 CGI 程序
    wait(NULL);                       /* Parent waits for and reaps child */
    close(pfd[1]);
}
```

### TCP 网络编程流程

**客户端流程：**
1. `socket()` 创建套接字
2. `gethostbyname()` 获取服务器地址
3. `connect()` 建立连接
4. `send()`/`recv()` 数据收发
5. `close()` 关闭连接

**服务器端流程：**
1. `socket()` 创建套接字
2. `bind()` 绑定地址
3. `listen()` 监听连接
4. `accept()` 接受连接
5. `send()`/`recv()` 数据收发
6. `close()` 关闭连接

### 重点知识

1. **套接字**是网络通信的核心抽象，包含 IP 地址和端口号
2. **TCP 连接**是一个文件描述符，可用 `read`/`write` 操作
3. **字节序**转换是网络编程的关键细节（`htons`/`ntohs`/`htonl`/`ntohl`）
4. **Web 服务器**的核心是解析 HTTP 请求行，根据请求类型返回静态或动态内容
5. **CGI 程序**通过管道接收参数，输出 HTML 内容到标准输出（被重定向到网络连接）

# 第9讲 并发网络服务器

## 1. 概念引入

迭代式网络服务器很难用于实际应用中，因为它们一次只能为一个客户端提供服务。改进方法是设计成**并发网络服务器**，让它创建一个单独的逻辑流服务每个客户端。使用应用级并发的应用程序称为**并发程序（Concurrent Program）**。

操作系统提供了三种基本的构造并发程序的方法：
1. **多进程（Multi-Process）**
2. **I/O 多路复用（I/O Multiplexing）**
3. **线程（Thread）**

本章用实例介绍这三种不同的并发编程技术。

### 本章学习目标

1. 理解基于多进程、I/O 多路复用、多线程、预线程化的并发网络服务器基本结构
2. 掌握基于多进程、多线程、预线程化进行并发网络服务器编程的基本方法

---

## 2. 核心概念：基于多进程的并发编程

### 2.1 基本思想

每次创建一个连接，就创建一个子进程与客户端通信。

### 2.2 程序代码（togglesp.c）

```c
void sigchld_handler(int sig) {
    while (waitpid(-1, 0, WNOHANG) > 0);
    return;
}

int main(int argc, char **argv)
{
    int listen_sock, conn_sock, port;
    socklen_t clientlen = sizeof(struct sockaddr_in);
    struct sockaddr_in clientaddr;
    if (argc != 2) { ... }
    port = atoi(argv[1]);
    signal(SIGCHLD, sigchld_handler);
    listen_sock = open_listen_sock(port);
    while (1) {
        conn_sock = accept(listen_sock, (SA *)&clientaddr, &clientlen);
        if (fork() == 0) {
            close(listen_sock);
            toggle(conn_sock);      /* Child process services client */
            close(conn_sock);
            exit(0);
        }
        close(conn_sock);
    }
}
```

**关键点：**
- 使用 `signal(SIGCHLD, sigchld_handler)` 注册信号处理函数，避免僵尸进程
- `sigchld_handler` 中使用 `waitpid(-1, 0, WNOHANG)` 非阻塞地回收子进程
- 子进程关闭 `listen_sock`，只使用 `conn_sock` 与客户端通信
- 父进程关闭 `conn_sock`，继续监听新的连接

### 2.3 测试

```bash
$ gcc togglec.c toggle.c -o togglec -L. -lwrapper
$ gcc togglesp.c toggle.c -o togglesp -L. -lwrapper
```

在三个不同的终端窗口执行程序：第一个窗口运行 `togglesp`，第 2、3 个窗口运行 `togglec`。

### 2.4 优缺点

**特点：** 父子共享打开文件表，但不共享用户地址空间。

**优点：**
- 每个进程都有独立的地址空间，进程间不会相互影响
- 有较好的可靠性和安全性

**缺点：**
- 进程间共享状态信息变得麻烦
- IPC 机制开销很高，进程间数据共享低效

## 3. 核心概念：基于线程的并发编程

### 3.1 基本思想

每次创建一个连接，就创建一个线程负责与客户端通信，降低创建、撤销进程的开销。

### 3.2 基于线程的并发 toggle 服务器（togglest.c）

```c
int main(int argc, char **argv)
{
    int listen_sock, *conn_sock_p, port;
    socklen_t clientlen = sizeof(struct sockaddr_in);
    struct sockaddr_in clientaddr;
    pthread_t tid;
    if (argc != 2) { ... }
    port = atoi(argv[1]);
    listen_sock = open_listen_sock(port);
    while (1) {
        conn_sock_p = malloc(sizeof(int));
        *conn_sock_p = accept(listen_sock, (SA *)&clientaddr, &clientlen);
        pthread_create(&tid, NULL, serve_client, conn_sock_p);
    }
}

void *serve_client(void *vargp)
{
    int conn_sock = *((int *)vargp);
    pthread_detach(pthread_self());
    free(vargp);
    toggle(conn_sock);
    close(conn_sock);
    return NULL;
}
```

**关键点：**
- 使用 `malloc` 为每个连接动态分配内存存储连接描述符，避免竞争条件
- 使用 `pthread_detach` 分离线程，使其结束后自动回收资源
- 线程使用 `free` 释放动态分配的内存

### 3.3 测试

```bash

```

## 4. 核心概念：基于预线程化的并发服务器

### 4.1 基本思想

预先创建一批**工作者线程（Worker Thread）**，每次建立一个连接，工作者线程就领取一个任务，负责与一个客户端通信。以消除服务器运行过程中创建、撤销线程的开销。

### 4.2 任务池定义（task_pool.c、task_pool.h）

采用**生产者/消费者模型（Producer/Consumer Model）**：
- 代表任务的连接是数据
- 缓冲区是数组 `socks`
- `inpos`、`outpos` 分别是缓冲区写入、读出指针
- `mutex`：为互斥信号量
- `avail`、`ready` 是同步信号量

```c
typedef struct {
    int *socks;         /* Buffer array */
    int cnt;            /* Maximum number of cell */
    int inpos;          /* buf[inpos] is first available cell */
    int outpos;         /* buf[outpos] is first item */
    sem_t mutex;        /* Protects accesses to socks */
    sem_t avail;        /* Counts available cells */
    sem_t ready;        /* Counts ready items */
} task_pool_t;
```

### 4.3 缓冲区初始化代码

```c
void task_pool_init(task_pool_t *tp, int n)
{
    tp->socks = Calloc(n, sizeof(int));
    tp->cnt = n;                     /* socks holds max of n items */
    tp->inpos = tp->outpos = 0;     /* Empty socks iff inpos == outpos */
    sem_init(&tp->mutex, 0, 1);     /* Binary semaphore for locking */
    sem_init(&tp->avail, 0, cnt);   /* Initially, socks has cnt empty cell */
    sem_init(&tp->ready, 0, 0);     /* Initially, socks has zero data items */
}
```

### 4.4 读写缓冲区代码

**插入任务（task_insert）：**

```c
void task_insert(task_pool_t *tp, int item)
{
    sem_wait(&tp->avail);           /* Wait for available cell */
    sem_wait(&tp->mutex);          /* Lock the shared variable tail pointer */
    tp->socks[tp->inpos] = item;   /* Insert the item */
    tp->inpos = (tp->inpos + 1) % (tp->cnt);  /* adjust tail point */
    sem_post(&tp->mutex);          /* Unlock the buffer */
    sem_post(&tp->ready);          /* Announce available item */
}
```

### 4.5 预线程化服务器代码（togglest_pre.c）

```c
int main(int argc, char **argv)
{
    int i, listen_sock, conn_sock, port;
    socklen_t clientlen = sizeof(struct sockaddr_in);
    struct sockaddr_in clientaddr;
    pthread_t tid;
    if (argc != 2) { ... }
    port = atoi(argv[1]);
    task_pool_init(&tp, SBUFSIZE);
    listen_sock = open_listen_sock(port);
    for (i = 0; i < NTHREADS; i++)  /* Create worker threads */
        pthread_create(&tid, NULL, serve_client, NULL);
    while (1) {
        conn_sock = accept(listen_sock, (SA *)&clientaddr, &clientlen);
        task_insert(&tp, conn_sock);    /* Insert conn_sock in task pool */
    }
}

void *serve_client(void *vargp)
{
    pthread_detach(pthread_self());
    while (1) {
        int conn_sock = task_remove(&tp);
        toggle(conn_sock);
        close(conn_sock);
    }
}
```

**关键点：**
- 主线程预先创建 `NTHREADS` 个工作线程
- 主线程只负责接受连接并插入任务池
- 工作线程从任务池中取出连接进行处理
- 使用信号量实现生产者/消费者同步

## 5. 总结复盘

### 三种并发方案对比

| 方案 | 基本思想 | 优点 | 缺点 |
|------|----------|------|------|
| 多进程 | 每个连接创建一个子进程 | 独立地址空间，可靠安全 | IPC 开销高，共享状态麻烦 |
| 多线程 | 每个连接创建一个线程 | 共享地址空间，开销小 | 需要处理线程同步 |
| 预线程化 | 预先创建工作者线程池 | 消除线程创建/撤销开销 | 实现复杂，需要任务池管理 |

### 关键术语回顾

| 术语 | 英文 | 说明 |
|------|------|------|
| 并发程序 | Concurrent Program | 使用应用级并发的应用程序 |
| 工作者线程 | Worker Thread | 预先创建的处理任务的线程 |
| 任务池 | Task Pool | 管理待处理连接的缓冲区 |
| 生产者/消费者模型 | Producer/Consumer Model | 主线程生产任务，工作者线程消费任务 |
| 线程分离 | Thread Detach | 线程结束后自动回收资源 |
| 信号量 | Semaphore | 用于线程间同步的计数器 |

### 重点知识

1. **多进程并发**通过 `fork()` 为每个连接创建子进程，使用 `signal(SIGCHLD, handler)` 避免僵尸进程
2. **多线程并发**通过 `pthread_create()` 为每个连接创建线程，使用 `pthread_detach()` 分离线程
3. **预线程化**预先创建线程池，使用**生产者/消费者模型**管理任务
4. **任务池**使用三个信号量（`mutex`、`avail`、`ready`）实现同步
5. 三种方案的核心区别在于**资源隔离程度**和**创建/撤销开销**的权衡
