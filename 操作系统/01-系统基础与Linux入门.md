# 第1讲 系统运行机制

## 1. 概念引入

操作系统需要硬件支持才能实现多进程并发运行和系统保护。核心问题有两个：

**运行机制支持**：多个进程（如进程A和进程B）的代码如何交替执行？如何保证结果正确？用户进程如何调用操作系统内核功能（如 `write()` 函数）？

**系统保护支持**：如何防止进程A、B相互干扰？如何保护操作系统内核不被恶意用户进程破坏？如何防止用户进程绕过操作系统内核直接操作资源？

**类比**：可以将CPU想象成一个"接力棒"，多个进程轮流拿到接力棒（CPU控制权）执行，中断机制就是传递接力棒的信号。

## 2. 核心公式

### 2.1 中断机制

**中断响应流程**（硬件完成）：
1. 保存当前PC（程序计数器，即断点）和机器状态到栈中
2. 关中断
3. 从数据总线读取中断类型号
4. 根据中断类型号跳转到对应的中断服务程序（ISR）

**进程切换公式**：
- 中断发生 → CPU保存进程A断点 → OS接管 → 调度更高优先级进程 → 中断返回到进程B断点

### 2.2 地址变换机制（MMU）

**实地址模式（无MMU）**：
- 程序地址 = 物理地址
- CPU直接将指令中的存储地址送到地址总线
- **无法实现进程隔离**：进程P1可以通过指针直接修改OS或进程P2的内存

**保护地址模式（有MMU）**：
- 程序地址（逻辑地址/虚拟地址VA）≠ 物理地址（PA）
- 程序地址空间：低地址为用户空间，高地址为内核空间
- MMU负责将逻辑地址转换为物理地址
- 每个进程有不同的地址变换函数，只能访问映射到自身的物理区域

### 2.3 CPU工作模式

**双模式保护**：
| 模式 | 权限 | 可执行指令 |
|------|------|-----------|
| 内核模式（系统态/核心态/管态） | 不受限制 | 任何指令（特权+非特权） |
| 用户模式（用户态/目态） | 受严格限制 | 仅非特权指令 |

**特权指令 vs 非特权指令**：
- 特权指令：涉及系统资源分配和管理，如修改页表、I/O指令、停机、修改PSW等
- 非特权指令：内存访问、通用寄存器操作、算术逻辑运算、子程序调用等

**模式切换规则**：
- 用户态 → 内核态：**只能通过中断响应**（interrupt、trap、异常）
- 内核态 → 用户态：通过中断返回（IRET指令）或直接修改PSW寄存器
- 函数调用（call/BL）**不能**改变CPU工作模式

### 2.4 系统调用机制

**系统调用本质**：通过特殊的软中断指令间接调用内核功能
- x86 CPU使用 `int` 指令，ARM CPU使用 `SWI` 指令
- Linux系统调用中断号为 `0x80`

**系统调用执行流程**：
1. 用户程序调用C库封装函数（如 `write()`）
2. 封装函数执行传送指令传递参数（通过寄存器EAX、EBX、ECX、EDX等）
3. 执行 `int $0x80` 陷入内核
4. `system_call()` 根据系统调用号查跳转表
5. 执行对应的内核函数（如 `sys_write()`）
6. 完成服务后返回用户程序

### 2.5 时钟中断

时钟中断的三个方面作用：
1. 驱动CPU指令执行（CPU的脉搏）
2. 维护计算机时间、日历
3. 为进程时间片调度和定时事件提供依据（操作系统的脉搏）

### 2.6 地址空间划分

**32位Linux**：用户空间3GB + 内核空间1GB
**32位Windows**：用户空间2GB + 内核空间2GB

**OS保护机制**：
- 页表中U/S位标识页面为内核态页面或用户态页面
- CPU模式位（如IOPL）标识当前CPU特权模式
- 核心态可访问所有页面，用户态仅能访问用户态页面

## 3. 方法速通

1. **识别中断类型**：外中断（I/O中断、时钟中断）由外部设备引发；内中断/异常由CPU执行指令时产生（如缺页异常、算术溢出）
2. **判断进程切换时机**：时钟中断中断当前进程，OS进行调度，将CPU分配给另一进程
3. **系统调用识别**：用户态调用标准C库函数（如 `open()`、`read()`），底层通过 `int $0x80` 陷入内核执行对应的 `sys_xxx()`
4. **MMU地址隔离**：每个进程有独立的地址变换函数，只能访问自身映射的物理区域
5. **模式切换判断**：用户态调用函数call不能切到内核态；只有中断（含软中断int指令）才能从用户态切到内核态

## 4. 典型示例

**示例1：多进程并发执行场景**

三个进程P1、P2、P3在OS控制下并发执行：
- 时钟中断每10ms发生一次，OS检查当前进程剩余时间片，进行进程调度
- 进程调用系统调用（如 `open()`）时，OS执行内核函数 `sys_open()` 完成操作
- 进程执行I/O操作等待事件时，OS将CPU分配给其他进程
- 进程结束（如执行 `exit()`）时，OS重新分配CPU

**示例2：printf的完整调用链**

```
用户程序 main() → C库函数 printf() → 系统调用封装函数 write() → int $0x80 → system_call() → sys_write()
```

**示例3：write系统调用的参数传递**

```asm
movl  $4, %eax      # 系统调用号4送EAX
movl  $1, %ebx      # 文件描述符1（stdout）送EBX
movl  $string, %ecx # 字符串首地址送ECX
movl  $14, %edx     # 字符串长度送EDX
int   $0x80         # 执行系统调用
```

## 5. 真题/实战

**常见考试题目类型**：

1. 什么是逻辑地址、物理地址？什么是内核空间、用户空间？
2. MMU的功能是什么？如何实现进程隔离？
3. 什么是特权指令？什么是非特权指令？各有什么特点？
4. CPU的两种工作模式各有什么特点？如何利用工作模式限制用户程序权限？
5. 使CPU从用户模式切换到核心模式/从核心模式切换到用户模式有哪些方法？
6. 时钟中断对操作系统工作有何作用？
7. 什么叫中断处理中的保存现场操作？

**判断题示例**：以下哪些指令执行时不可能导致CPU从用户态变为内核态？
- `DIV R0,R1`：不会（非特权指令）
- `INT n`：**会**（产生软中断，切换到内核态）
- `NOT R0`：不会（非特权指令）
- `MOV R0,addr`：不会（非特权指令）

> 提示：`open()`、`close()`、`read()`、`write()`、`fork()`、`exec()`、`wait()`、`exit()`、`brk()`、`ioctl()`

**练习2**：以下指令中哪些是特权指令（只能在核心态执行）？
1) 读时钟 2) 写程序计数器 3) 执行int指令 4) 关中断 5) 设置模式为核心态 6) I/O操作 7) 修改页表 8) 设置日历钟 9) 重新引导 10) 读程序状态字 11) 禁止中断

## 7. 总结复盘

**易错点**：
- 中断响应（硬件完成）与中断处理（软件完成）是两个不同阶段
- 函数调用（call/BL）不能改变CPU工作模式，只有中断可以
- MMU在用户态阻止访问内核态页面，但内核态可以访问所有页面
- 系统调用通过软中断指令（int/SWI）实现，不是直接函数调用
- 断点是指被中断打断时的下一条指令地址，不是当前指令地址

**关键要点思维导图**：
- 硬件支持五大部分：中断机制 → 时钟中断 → MMU地址变换 → CPU双模式 → 系统调用
- 中断机制：外中断（I/O、时钟）+ 内中断（异常）+ 软中断（系统调用）
- MMU：逻辑地址 → 物理地址映射，实现进程隔离
- CPU模式：内核态（R0-R2/0b10001-0b11111）与用户态（R3/0b10000）
- 系统调用流程：用户程序 → C库封装 → 软中断 → system_call → sys_xxx
- 时钟中断驱动进程调度，实现多任务并发

# 第2讲 Linux目录结构与文件操作

## 1. 概念引入

Linux是一种类UNIX操作系统，继承了UNIX强大的功能和性能。与Windows不同，Linux没有盘符概念，所有文件都挂载在一棵以 `/` 为根目录的树上。

**为什么学Linux？**
- 世界上最快的10台超级计算机全部运行Linux
- 开源免费，开发工具丰富（gcc、java、apache等）
- 与UNIX兼容，学会操作Linux即会操作UNIX

## 2. 核心公式

### 2.1 Linux目录结构规范

以 `/` 为根目录的树形结构，主要目录有统一规范：

| 目录 | 用途 |
|------|------|
| `/home` | 普通用户家目录 |
| `/root` | root用户的家目录 |
| `/usr` | 系统文件（可只读挂载） |
| `/var` | 动态增长文件（读写挂载） |
| `/etc` | 系统配置文件 |
| `/tmp` | 临时文件 |

### 2.2 路径表示

- **绝对路径**：从根目录 `/` 开始的完整路径，如 `/home/can/work/f1`
- **工作目录**：每个Terminal和进程设置的工作目录，初始为用户家目录
- **相对路径**：相对于工作目录的路径

**特殊目录符号**：
- `.`：当前工作目录
- `..`：上一层目录
- `~`：当前用户的家目录（如 `~guest` 表示guest的家目录）
- `-`：前一个工作目录

### 2.3 文件权限模型

`ls -l` 显示的9位权限结构：`rwxrwxrwx`

| 位置 | 含义 |
|------|------|
| 第1位 | 文件类型（-普通文件，d目录，l链接等） |
| 第2-4位 | 所有者权限（r读/w写/x执行） |
| 第5-7位 | 同组用户权限 |
| 第8-10位 | 其他用户权限 |

**普通文件权限**：r=读内容，w=改内容，x=执行程序
**目录文件权限**：r=列目录内容，w=增删文件，x=进入目录

**数字表示法**：r=4, w=2, x=1
- `755` = rwxr-xr-x
- `664` = rw-rw-r--

### 2.4 文件操作命令格式

```
$ 命令名 选项 参数1 参数2 ...
```

## 3. 方法速通

**目录操作**：
- `cd` 切换目录，`pwd` 显示当前路径
- `mkdir` 创建目录，`rmdir` 删除空目录，`rm -rf` 删除非空目录
- `ls` 查看目录，常用 `-a`（全部文件）、`-l`（长格式）、`-A`（除.和..）

**文件操作**：
- `cp` 复制：`-r` 递归复制目录，`-f` 强制，`-l` 硬链接，`-s` 符号链接
- `mv` 移动/重命名：`-f` 强制，`-i` 覆盖前询问
- `rm` 删除：`-r` 递归，`-f` 强制

**权限修改**：
- `chmod 777 file`：数字方式修改
- `chmod +x file`：添加执行权限
- `chmod u+w file`：所有者添加写权限

**文件查看**：
- `cat`、`more`、`less`：查看文本文件内容
- `head`、`tail`：查看文件头/尾

**打包压缩**：
- 打包：`tar -cvf archive.tar dir`
- 解包：`tar -xvf archive.tar`
- 打包压缩：`tar -czvf archive.tar.gz dir`
- 解压解包：`tar -zxvf archive.tar.gz`

## 4. 典型示例

**示例1：目录操作完整流程**
```bash
$ cd /tmp
$ pwd
/tmp
$ rm -rf *                    # 清空目录
$ mkdir test                  # 创建目录test
$ mkdir test1 test/sub test2  # 一次创建多个目录
$ ls .
test test1 test2
$ rmdir test1                 # 删除空目录
$ rm -rf test                 # 删除非空目录
```

**示例2：ls命令常用选项**
```bash
$ ls -a          # 列出所有文件，包括隐藏文件
$ ls -A          # 列出所有文件，不包括 . 和 ..
$ ls -l          # 长格式显示（权限、大小、时间等）
$ ls -F          # 显示文件类型标记（/目录，*可执行，|管道）
$ ls -R          # 递归显示子目录内容
$ ls -i          # 显示inode编号
```

**示例3：文件复制与链接**
```bash
$ cp ~/.bashrc bashrc                  # 复制单个文件
$ cp -rf /etc/ /tmp                    # 复制整个目录
$ cp -s bashrc bashrc_slink            # 创建符号链接
$ cp -l bashrc bashrc_hlink            # 创建硬链接
$ ln -s bashrc bashrc_slink            # 等效的符号链接命令
$ ln bashrc bashrc_hlink               # 等效的硬链接命令
```

**示例4：权限修改**
```bash
$ chmod 777 f52         # 将f52权限改为rwxrwxrwx
$ chmod +x test.sh      # 添加执行权限
$ chmod u+w f521        # 所有者添加写权限
$ chmod a-w f522        # 去掉所有用户写权限
```

## 6. 巩固练习

**练习1**：简述Linux目录结构设计的好处。

## 7. 总结复盘

**易错点**：
- Linux没有盘符概念，所有内容在根目录 `/` 下
- `rmdir` 只能删除空目录，删除非空目录用 `rm -rf`
- 文件权限对普通文件和目录文件含义不同（如x权限）
- 硬链接指向同一inode，删除原文件后硬链接仍可访问
- `rm -rf` 是危险命令，删除前务必确认路径正确
- 命令选项顺序一般不影响结果，但参数顺序可能影响（如 `cp src dst`）

**关键要点思维导图**：
- 目录结构：以 `/` 为根的树形结构，无盘符
- 路径：绝对路径（完整路径） vs 相对路径（基于工作目录）
- 特殊符号：`.`（当前） `..`（上级） `~`（家目录） `-`（上一个目录）
- 权限模型：`rwx` × 三组（所有者/同组/其他），数字表示法
- 核心命令：cd/pwd/mkdir/rmdir/ls/cp/mv/rm/chmod/tar
- 文件操作：复制(cp)、移动(mv)、删除(rm)、查看(cat/more/less)、压缩(tar)

# 第3讲 Linux环境C编程

## 1. 概念引入

在Linux环境下进行C语言程序开发，需要掌握从编辑、编译、链接到执行的完整流程。理解编译过程的四个阶段（预处理、编译、汇编、链接）对于调试和优化程序至关重要。

**核心问题**：
- 从 `.c` 源文件到可执行程序经历了什么？
- 如何诊断和处理编译错误和运行错误？
- 如何使用系统库函数提高开发效率？
- 如何进行多文件项目编译？

## 2. 核心公式

### 2.1 C程序编译四阶段

```
.c/.cpp文件 → [预处理] → .i文件 → [编译] → .s文件 → [汇编] → .o文件 → [链接] → 可执行文件
```

**各阶段功能**：
1. **预处理**（Preprocessing）：处理所有以 `#` 开头的语句（头文件展开、宏替换、条件编译）
2. **编译**（Compilation）：将C/C++源程序翻译成汇编语言程序
3. **汇编**（Assembly）：将汇编程序翻译成目标代码文件（二进制）
4. **链接**（Linking）：将多个目标代码文件和系统库链接，产生可执行文件

**gcc命令选项**：
- `gcc hello.c`：四阶段一次性完成，生成可执行文件
- `gcc -E hello.c -o hello.i`：仅预处理，输出 `.i` 文件
- `gcc -S hello.c -o hello.s`：仅编译，输出 `.s` 汇编文件
- `gcc -c hello.c -o hello.o`：仅汇编，输出 `.o` 目标文件
- `gcc hello.c -o hello`：指定输出文件名

### 2.2 静态库 vs 动态库

**静态库（.a文件）**：
- 编译时将库代码复制到可执行文件中
- 可执行文件较大，但运行时不需要库文件
- 创建：`ar rc libmycalc.a aver.o sum.o`
- 使用：`gcc libtest.c -Bstatic -L. -lmycalc -o libtesta`

**动态库（.so文件）**：
- 编译时仅记录库引用，运行时动态加载
- 可执行文件较小，但运行时需要库文件
- 创建：`gcc -shared -o libmycalc.so aver.o sum.o`
- 使用：`gcc libtest.c -L. -o libtestso -lmycalc`
- 注意：需要设置 `LD_LIBRARY_PATH` 环境变量

### 2.3 错误分类

**编译错误**：
- Error：致命错误，编译失败，不产生输出文件
- Warning：警告错误，编译仍可成功，产生输出文件

**运行错误**：
- 程序运行结果不正确
- 程序运行崩溃（crash）

### 2.4 系统调用错误处理

**基本方法**：
```c
if (fd == -1) {
    fprintf(stderr, "file open error: %s\n", strerror(errno));
}
```

**Randy Bryant包装函数方法**：
- 包装函数名：原函数名首字母大写（如 `Open`、`Read`、`Write`）
- 包装函数检查错误并调用 `perror()` 输出错误信息
- 使用：`#include "wrapper.h"` 并链接 `-lwrapper`

### 2.5 环境变量与命令行参数

**环境变量读取**：
```c
#include <stdlib.h>
char *s = getenv("PWD");      // 读取系统环境变量
export PHONE=0769-22861112     // 创建自定义环境变量
```

**命令行参数**：
```c
int main(int argc, char *argv[]) {
    // argc：参数个数（包括程序名本身）
    // argv：参数字符串数组
}
```

## 3. 方法速通

**快速诊断编译错误**：
1. 查看错误信息中的文件名和行号
2. 常见错误类型：变量无定义、重复定义、缺少括号/分号、找不到文件、中文字符
3. 链接阶段错误通常是找不到库文件或函数未定义

**快速调试运行错误**：
1. 使用 `printf` 输出关键变量值
2. 使用 `gdb` 或 `ddd` 调试器
3. 使用 `assert()` 断言检查程序状态
4. 设计错误处理机制（try...catch、错误处理包装库）

**多文件编译技巧**：
1. 先将每个 `.c` 文件编译成 `.o` 文件
2. 再链接所有 `.o` 文件生成可执行文件
3. 或一条gcc命令编译所有源文件
4. 大型项目使用 `makefile` 管理

## 4. 典型示例

**示例1：完整编译流程演示**

```bash
# 编辑源文件
$ gedit hello.c &

# 编译执行
$ gcc hello.c        # 生成a.out
$ ./a.out            # 输出：hello world

# 查看编译中间文件
$ gcc -E hello.c -o hello.i    # 预处理
$ gcc -S hello.c -o hello.s    # 编译
$ gcc -c hello.c -o hello.o    # 汇编
$ nm hello.o                   # 查看符号表
```

**示例2：多文件项目编译**

```bash
# 方法1：分步编译
$ gcc -c sum.c
$ gcc -c aver.c
$ gcc -c libtest.c
$ gcc -o libtest sum.o aver.o libtest.o

# 方法2：一条命令编译
$ gcc sum.c aver.c libtest.c -o libtest
```

**示例3：静态库创建与使用**

```bash
# 编译库函数源程序
$ gcc -c -o aver.o aver.c
$ gcc -c -o sum.o sum.c

# 创建静态库
$ ar rc libmycalc.a aver.o sum.o

# 编译应用程序（静态链接）
$ gcc libtest.c -Bstatic -L. -lmycalc -o libtesta

# 执行
$ ./libtesta
```

**示例4：动态库创建与使用**

```bash
# 编译源程序（产生位置无关代码）
$ gcc -c -fPIC aver.c -o aver.o
$ gcc -c -fPIC sum.c -o sum.o

# 创建动态库
$ gcc -shared -o libmycalc.so aver.o sum.o

# 编译应用程序
$ gcc libtest.c -L. -o libtestso -lmycalc

# 设置动态库路径
$ export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:.

# 执行
$ ./libtestso
```

## 7. 总结复盘

**易错点**：
- 预处理阶段展开头文件，如果头文件路径错误会导致"找不到文件"错误
- 中文全角字符、标点符号会导致编译错误，注意使用英文字符
- 链接阶段错误通常是函数未定义或找不到库文件
- 动态库使用时需要设置 `LD_LIBRARY_PATH` 环境变量
- `qsort` 函数的比较函数返回值定义：负数表示a<b，0表示a==b，正数表示a>b
- 系统调用失败时返回 `-1`，需要检查返回值并处理错误

**关键要点思维导图**：
- 编译流程：预处理(.i) → 编译(.s) → 汇编(.o) → 链接(可执行文件)
- gcc选项：-E(预处理) -S(编译) -c(汇编) -o(输出文件名)
- 库类型：静态库(.a) 编译时链接 | 动态库(.so) 运行时加载
- 错误处理：编译错误(Error/Warning) + 运行错误(结果错误/崩溃)
- 调试方法：printf输出 + gdb调试 + assert断言 + 包装函数
- 项目管理：多文件编译 + makefile管理
- 常用库函数：字符串处理、数学运算、时间函数、数据结构算法
